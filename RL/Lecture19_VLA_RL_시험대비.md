# Lecture 19 — VLA 모델을 강화학습으로 개선하기 (서술형 시험 대비 정리)

> **이 강의 한 문장 요약**
> VLA는 "규모를 키운 모방학습"이라 시연(demonstration)을 넘어설 수 없다. 그래서 **사전학습 → 모방(SFT) → 강화학습(RL)** 의 3단계 패러다임에서 마지막 RL 단계를 붙여, 보상으로부터 학습하게 함으로써 시연자보다 더 잘하는 에이전트로 만든다.

> **서술형 대비 사용법**
> 각 절은 ① 핵심 개념 → ② **왜 이렇게 하는가(제안배경)** → ③ **무엇이 문제인가(한계·도전)** → ④ **어디로 확장되는가(방향)** 순서로 읽으면 됩니다. 문서 맨 끝의 **예상 서술형 문제 + 답안 골격**을 먼저 훑고, 본문으로 근거를 채우는 방식이 효율적입니다.

## 0. 전체 그림 한눈에 보기

이 강의는 코스를 닫는 강의로, 앞의 두 흐름을 합칩니다.

- **VLA 강의에서**: VLA = 규모를 키운 모방학습. 시연을 복사하므로 **시연자를 넘어설 수 없고**, 더 나은 전략을 발견할 수 없으며, **자기 실수에서 회복할 방법이 없다.**
- **DPO/GRPO 강의에서**: 이제 우리는 시연뿐 아니라 **보상·선호로부터 모델을 개선하는 도구**를 갖고 있다.
- **자연스러운 다음 단계**: RL로 VLA를 미세조정(fine-tuning)해서 **시연을 넘어서게** 만든다.

코스 전체의 아크(arc):
**모방(BC·DAgger·IRL·GAIL) → VLA(규모를 키운 모방) → DPO/GRPO(선호·보상으로 정렬) → VLA + RL(시연을 능가)**

> 관통하는 단 하나의 아이디어: **"신호(signal)로부터 행동을 학습한다."** 그 신호가 보상이든, 시연이든, 선호든 — 모방·역강화학습·VLA·DPO·GRPO·VLA+RL은 모두 이 하나의 주제의 변주다.

## 1. 왜, 그리고 어떻게 VLA에 RL을 더하는가

### 1.1 모방학습 기반 VLA의 근본적 한계 (제안배경)

모방학습(imitation)은 **데이터 속 평균적 행동을 그대로 복사**한다. 이때 실수까지 함께 복사하며, 그 행동이 최적인지 **결코 묻지 않는다.** 여기서 세 가지 근본 한계가 나온다.

1. **시연자 초과 불가**: 데이터에 없는 더 나은 전략을 발견할 수 없다. 성능의 상한은 시연 품질이다.
2. **회복 능력 부재**: 자기 실수의 결과를 경험할 길이 없으므로, 새로운(=시연에 없던) 상태에서 어떻게 복구할지 모른다.
3. **수동적 모방자**: 환경과 상호작용하며 "무엇이 성공으로 이어지는가"를 능동적으로 탐색하지 못한다.

→ 이 한계를 깨려면 **결과(outcome)로부터 학습**하는 메커니즘, 즉 강화학습이 필요하다.

### 1.2 구체적 예시 — "물 따르기"로 보는 복사의 한계

- 모방으로 컵에 물 따르기를 학습한 VLA가 있다.
- 실제 시도 중 **약간 더 기울여서** 컵이 미끄러지기 시작한다 — 어떤 시연에도 없던 상황.
- 복사만 배웠으므로 이 새 상태에 대한 **대응이 없고 실패**한다.
- 시연자는 정확히 이 실수를 한 적이 없으니, 데이터에는 **모방할 복구 동작이 존재하지 않는다.**
- RL이 있으면 로봇은 **시도하고, 실패하고, 낮은 보상을 느끼며, 스스로 교정 동작을 점진적으로 학습**할 수 있다.

> 서술형 포인트: 이 예시는 "왜 복사로 충분하지 않은가"를 보여주는 표준 사례. **분포 밖(out-of-distribution) 상태**와 **복구 데이터의 부재**라는 두 키워드로 설명할 것.

### 1.3 현대 VLA 학습 패러다임 — 3단계와 "능력의 상한"

| 단계 | 하는 일 | 학습 신호 | 도달 가능한 상한 |
|------|---------|-----------|------------------|
| **Pretrain** | 비전 + 언어 이해 | 인터넷 지식 | 일반 지식 (행동 접지 없음) |
| **SFT (imitation)** | 시연 모방 | 전문가 복사 | **시연 품질까지** |
| **RL** | 보상으로 개선 | 보상·시행착오 | **시연을 능가** |

- 앞의 **두 단계는 본질적으로 모방**이다 → 시연 품질을 넘을 수 없다(천장이 있다).
- **RL 단계에서 비로소 그 천장을 깬다.** 이 강의의 초점이 바로 여기.

### 1.4 RL이 VLA에 더해주는 네 가지

순수 모방이 줄 수 없는 네 가지를 RL이 제공한다.

1. **시연 초월(Surpassing demos)**: 시연 집합에 없는 더 나은 전략을 발견.
2. **교정 행동(Corrective behavior)**: 실수의 결과를 경험하며 복구를 학습.
3. **일반화(Generalization)**: RL로 튜닝된 정책은 SFT보다 **분포 밖 상황을 더 잘 처리**하는 경향.
4. **장기 과제 수행(Long-horizon skill)**: 과제 전체에 대한 보상이 여러 단계를 목표를 향해 연결(chain)하도록 도움.

> 핵심: RL은 **수동적 모방자를 → 과제에서 능동적으로 개선하는 에이전트로** 바꾼다.

### 1.5 RL이 모방을 이길 수 있는 이유

- 모방은 데이터의 평균 행동을 실수까지 복사하고, 최적성을 **묻지 않는다.**
- RL은 **"어떤 행동이 실제로 높은 보상으로 이어지는가"** 를 묻고 그것을 강화한다.
- 결과로부터 배우므로, 성공을 좌우하는 **소수의 결정적 결정(critical decisions)에 집중**할 수 있다.
- **강한 증거**: RL 미세조정이 SFT 단독보다 **훨씬 나은 분포 밖(OOD) 성능**을 준다.
- 다만 대형 VLA에 RL을 적용하는 것은 **고유한 도전**들을 동반한다(아래 2장).

## 2. VLA에 RL을 적용할 때의 세 가지 도전과제

> 서술형에서 가장 잘 나올 핵심. "왜 LLM처럼 그냥 RL을 못 붙이는가?"에 대한 세 가지 답.

### 2.1 도전 1 — 보상은 어디서 오는가?

게임은 내장된 점수가 있지만, **로봇 과제에는 명백한 보상 신호가 없다.** VLA 연구의 흔한 선택지 3가지:

| 보상 방식 | 정의 | 장점 | 단점 |
|-----------|------|------|------|
| **Sparse success(희소 성공)** | 과제 완료 시 1, 아니면 0 | 단순·명확 | **학습이 어려움** (신용 할당 곤란, 신호 희박) |
| **Verifiable(검증 가능)** | 자동 확인 가능한 결과 (예: 물체를 잡았나/제대로 놓았나) | 자동 채점, 비교적 구조적 | 검증 항목 설계 필요 |
| **World-model(세계 모델)** | 학습된 모델이 결과를 예측·채점 | **실제 로봇 롤아웃 회피**(저렴·안전) | 모델 오차에 취약 |

> **핵심 난제**: 보상 설계가 중심적 실무 난점. **나쁜 보상은 정책이 과제를 푸는 대신 보상을 편법으로 게이밍(game)하게** 만든다. (→ 이후 "열린 과제"에서 다시 등장)

### 2.2 도전 2 — 행동 표현(Action Representation)이 중요하다

정책경사(policy gradient) RL은 정책이 행동에 부여하는 **확률 $\log \pi_\theta(a \mid o)$** 가 필요하다. 이게 계산 가능하냐가 행동 표현 방식에 달려 있다.

- **토큰 기반 VLA** (RT-2, OpenVLA)
  - 행동이 **이산 토큰**으로 표현됨 → LLM과 똑같이 확률 계산이 **쉽다.**
  - 그래서 **지금까지 대부분의 VLA+RL 방법이 정확히 이 자기회귀(autoregressive) 모델을 겨냥**한다.
- **Flow/Diffusion 기반 VLA** (pi-zero)
  - 행동이 **반복적 디노이징(iterative denoising)** 으로 생성됨 → 행동 우도(likelihood)가 **다루기 어려움(intractable).**
  - 표준 정책경사를 그대로 못 쓴다 → RL 적용이 막힘.
  - **pi-RL** 같은 최근 연구가 이 flow 기반 모델에 온라인 RL을 적용하는 **특수 기법**을 개발.

> 서술형 포인트: "토큰 기반 vs flow 기반" 난이도 차이는 **우도 계산 가능성**으로 설명. 토큰은 확률의 곱으로 바로 나오지만, flow/diffusion은 디노이징 궤적에 대한 적분이 필요해 닫힌 형태로 구하기 어렵다.

### 2.3 도전 3 — Online vs Offline RL

| 구분 | **Online RL** (환경과 상호작용) | **Offline RL** (고정 데이터셋에서 학습) |
|------|-------------------------------|----------------------------------------|
| 학습 원천 | **자신의 새 경험**에서 학습 | 환경 상호작용 **불필요** |
| 필요 자원 | 시뮬레이터 또는 실제 로봇 필요 | 로그 데이터만 있으면 됨 |
| 장점 | **데이터를 넘어설 수 있음** | **더 안전하고 저렴** |
| 단점 | 비용 큼, **sim-to-real 갭**, 안전 우려 | **분포 이동(distribution shift)에 취약** |

> 결론: **온라인은 데이터를 능가할 수 있으나 비싸고, 오프라인은 안전하나 제한적.** 그래서 **많은 방법이 둘을 혼합(blend)** 한다. (예: 뒤의 iRe-VLA가 RL 탐색 ↔ SFT를 번갈아 가는 것도 이 정신.)

## 3. DPO/GRPO와의 연결 + VLA에서의 RL 목적함수

### 3.1 지난 강의의 정렬 도구가 VLA로 그대로 이전된다

- **GRPO가 자연스럽게 맞는다**: 한 과제에 대해 **여러 행동 시퀀스를 샘플 → 채점 → 평균보다 나은 것을 강화.**
  - **그룹 상대 어드밴티지(group-relative advantage)는 크리틱(critic)이 필요 없다.** → 가치망(value network)이 비싼 **대형 VLA에 특히 매력적.**
- **DPO 스타일**: 로봇 행동에 대한 **선호(이 궤적이 저 궤적보다 낫다)** 가 있을 때 적합.

> 그래서 **언어 모델을 정렬했던 동일한 레시피가 이제 로봇 정책을 개선**한다.

**왜 "크리틱 불필요"가 큰 장점인가 (서술형 심화)**:
표준 액터-크리틱(PPO 등)은 분산을 줄이기 위해 별도의 **가치망 $V(s)$** 를 베이스라인으로 학습해야 한다. 대형 VLA에서는 비슷한 규모의 가치망을 추가로 학습하는 게 메모리·연산상 매우 비싸다. GRPO는 **그룹 평균을 베이스라인**으로 써서 학습된 가치함수 없이 이 문제를 우회한다. (뒤의 RIPT-VLA가 쓰는 RLOO도 "나머지 샘플들의 평균을 베이스라인으로" 쓰는 같은 정신.)

### 3.2 VLA 설정에서의 RL 목적함수 (수식)

동일 관측(observation)에 대해 **행동 시퀀스 그룹**을 샘플하고, 각각을 과제 보상으로 채점한 뒤, 그룹 내에서 표준화:

$$
A_i = \frac{r_i - \mathrm{mean}(r_1, \dots, r_G)}{\mathrm{std}(r_1, \dots, r_G)}
$$

그다음 VLA 정책을 업데이트하여 **어드밴티지가 높은 행동 시퀀스의 확률을 높인다.** — 정확히 **GRPO 아이디어를 로봇 행동에 적용**한 것.

> **결정적 통찰(서술형 단골)**: 보상 $r_i$ 가 이제 **인간 선호 대신 과제 성공**을 측정한다는 점만 다르고, **최적화 자체는 완전히 동일**하다. 이것이 LLM 정렬 → 로봇 정책 개선으로의 우아한 이전.

## 4. VLA RL 미세조정 방법론 — 세 갈래(families)

> 슬라이드 명시: 방법들은 "가볍게(kept light)" 골랐다 = **개별 방법의 디테일보다 "지형(landscape)과 테마"를 이해하는 것이 핵심.** 서술형도 이 테마 중심으로 준비.

| 갈래 | 한 줄 정의 | 핵심 trade-off |
|------|-----------|----------------|
| **Online RL** | 상호작용 후 개선 | 데이터 초과 가능 ↔ 비쌈/위험 |
| **Offline RL** | 로그 데이터에서 학습 | 안전·저렴 ↔ 분포 이동에 제한 |
| **Test-time RL** | 실행 시점에 최선의 행동 선택 (재학습 X) | 매우 저렴·안전 ↔ 새 기술은 못 배움 |

### 4.1 방법 (1) — 온라인 RL의 규모 확장과 안정화

- **SimpleVLA-RL**: GRPO 스타일 접근으로 **RL을 이용한 VLA 학습을 규모 확장.**
  - **핵심 발견**: 시연이 **소수만 있어도** RL이 성능을 끌어올림 → **거대한 시연 집합에 대한 의존을 줄임.**
  - (의의: 시연 수집은 로봇 학습의 병목이므로, 적은 데이터로 가능하다는 건 실용적으로 큼.)
- **iRe-VLA**: **RL 탐색 ↔ 지도학습(SFT)** 을 번갈아 수행.
  - 둘을 반복하면 **대형 VLA의 학습이 안정화**됨 — 순수 온라인 RL만으로는 학습이 **불안정(fragile)** 할 수 있기 때문. (SFT가 일종의 앵커/정규화 역할.)
- **테마**: **VLA 규모에서 온라인 RL을 실용적이고 안정적으로** 만들기.

### 4.2 방법 (2) — 새로운 목적함수와 Flow 모델

- **RIPT-VLA**: 단순한 **REINFORCE 계열 목적함수(RLOO, leave-one-out 베이스라인)** 를 쓰는 **상호작용형 사후학습(interactive post-training).**
  - 이미 학습된 VLA를 **가벼운 온라인 피드백**으로 정제(refine).
- **pi-RL**: **flow 기반 VLA(pi-zero 계열)** 에 온라인 RL을 도입.
  - diffusion·flow 모델의 RL을 막던 **다루기 어려운 행동 우도(intractable likelihood) 문제**를 해결.
- **테마**: **토큰 기반 VLA를 넘어 RL을 확장**하고, **더 단순한 목적함수**를 탐색.

### 4.3 방법 (3) — 실제 경험으로부터의 학습

- **pi*-0.6** (Physical Intelligence): **자기 경험으로부터 학습하도록 명시적으로 설계된** 최신 VLA.
  - 시연만 모방하는 대신, **과제를 직접 연습하고 그 결과를 학습 신호로** 사용해 개선.
- **장기 목표를 가리킴**: **운용할수록 계속 더 좋아지는 로봇** — 사람이 기술을 연습하듯이.
- **테마**: 루프를 닫아 **배치(deployment) 자체가 개선의 원천**이 되게 하기.

### 4.4 가벼운 선택지 — Test-Time RL

모든 방법이 모델을 재학습하지는 않는다. **Test-time RL은 행동하는 순간에 행동을 개선**한다.

- **아이디어**: 각 단계에서 VLA로부터 **여러 후보 행동을 샘플 → 가치 추정으로 순위 매김.**
- **최상위 행동만 실행** → **가중치를 바꾸지 않고도** 더 나은 선택.
- **장점**: 학습 루프가 없어 **저렴하고 안전.**
- **한계**: **진짜로 새로운 기술은 가르치지 못함** (기존 정책이 이미 만들 수 있는 후보 중에서 더 나은 것을 고를 뿐).
- **위치**: 전체 RL 미세조정이 **너무 비싸거나 위험할 때의 실용적 중간 지대.**

---

## 5. 열린 과제(Open Challenges)

VLA+RL은 유망하지만 **아직 초기**이고, 몇 가지 어려운 미해결 문제가 있다.

1. **보상 설계(Reward design)**: **게이밍되지 않으면서** 진짜 과제 성공을 포착하는 보상을 정의하기.
2. **Sim-to-real 갭**: 시뮬레이션에서 학습한 정책이 **실제 하드웨어에서 성능이 저하**되는 문제.
3. **비용과 안전(Cost & safety)**: 실세계 RL은 비싸고, **탐색 도중 안전하지 않을 수 있음.**
4. **Flow 기반 모델**: diffusion·flow VLA에 RL을 적용하는 것은 **여전히 해결 중.**

> 강조: 이들은 정확히 **이 코스의 RL 기초가 적용되는 종류의 문제들**이다. (서술형에서 "왜 어려운가"를 도전과제 2장과 연결해 설명하면 좋다 — 예: 보상 설계는 도전 1, flow 모델은 도전 2와 직결.)

---

## 6. 전체 코스를 한 그림으로

```
모방(Imitation)          VLA                  DPO / GRPO            VLA + RL
BC, DAgger,       →     규모를 키운      →     선호·보상으로     →    시연을
IRL, GAIL              모방                    정렬                   능가
```

- 코스의 아크: **전문가를 복사하는 법을 배우고 → 파운데이션 모델로 그것을 규모 확장하고 → 강화학습으로 그것을 넘어선다.**
- **모방은 에이전트에게 복사를 가르쳤고, 강화학습은 개선을 가르쳤다.**

---

## 7. 요약 (강의 자체 요약)

- VLA는 규모를 키운 모방이므로, **시연을 넘어 개선하려면 RL이 필요.**
- 현대 패러다임은 **사전학습 → SFT → RL 미세조정.**
- 핵심 도전은 **보상 설계, 행동 표현, 온라인 vs 오프라인 RL.**
- 지난 강의의 **GRPO·DPO가 그대로 이전**: 샘플한 행동 시퀀스를 채점하고 나은 것을 강화.
- 최근 방법들(**SimpleVLA-RL, iRe-VLA, RIPT-VLA, pi-RL, pi*-0.6**)이 RL로 VLA가 시연자를 능가할 수 있음을 보임.

# 📝 예상 서술형 문제 + 답안 골격

> 실제 답안 작성 시: **(1) 개념 정의 → (2) 왜 그런가(배경/원리) → (3) 한계/대안 → (4) 강의 다른 부분과 연결** 의 4박자로 쓰면 점수가 잘 나옵니다.

### Q1. 모방학습 기반 VLA의 근본적 한계를 설명하고, RL이 이를 어떻게 보완하는지 서술하라.
- 한계 3가지: 시연자 초과 불가(상한 = 시연 품질) / 자기 실수 회복 불가 / 수동적 모방자.
- 근거: 모방은 평균 행동을 실수까지 복사하며 최적성을 묻지 않음.
- RL의 보완: 결과로부터 학습 → 시연 초월·교정 행동·일반화·장기 과제 수행(네 가지).
- 예시로 마무리: "물 따르기 → 컵 미끄러짐" 사례로 OOD 상태 + 복구 데이터 부재 설명.

### Q2. 현대 VLA 학습의 3단계 패러다임을 설명하고, 각 단계의 성능 상한이 왜 다른지 논하라.
- Pretrain(인터넷 지식) / SFT(전문가 복사) / RL(보상으로 개선).
- 핵심 논지: **앞 두 단계는 본질이 모방 → 시연 품질이라는 천장이 있다.** RL만이 그 천장을 깬다.
- 왜 RL이 천장을 깨는가: 데이터가 아니라 **결과(보상)** 를 최적화하므로 데이터에 없는 전략을 발견.

### Q3. VLA에서의 보상 설계 문제를 설명하고, 세 가지 보상 방식을 비교하라.
- 전제: 로봇 과제엔 내장 점수가 없음 → 보상을 설계해야 함.
- 세 방식: Sparse success(단순하나 학습난) / Verifiable(자동채점, 구조적) / World-model(실로봇 롤아웃 회피, 모델오차 취약).
- 중심 난제: **나쁜 보상 → 정책이 과제 대신 보상을 게이밍.** (열린 과제와 연결)

### Q4. 토큰 기반 VLA와 flow/diffusion 기반 VLA에서 RL 적용 난이도가 다른 이유를 행동 표현 관점에서 설명하라.
- 정책경사는 $\log\pi_\theta(a\mid o)$ 가 필요.
- 토큰 기반(RT-2, OpenVLA): 이산 토큰 → 확률 계산 쉬움(LLM과 동일) → 대부분 방법이 이를 겨냥.
- flow/diffusion(pi-zero): 반복 디노이징 → 우도가 intractable → 표준 정책경사 적용 불가.
- 해결 시도: pi-RL이 flow 모델용 특수 기법 개발.

### Q5. Online RL과 Offline RL의 장단점을 비교하고, 왜 많은 방법이 둘을 혼합하는지 설명하라.
- 온라인: 새 경험 학습, 데이터 초과 가능 ↔ 비용·sim-to-real 갭·안전.
- 오프라인: 상호작용 불필요, 안전·저렴 ↔ 분포 이동 취약.
- 혼합 이유: 온라인의 "초월 능력"과 오프라인의 "안전·저비용"을 절충. 예시로 iRe-VLA(RL 탐색 ↔ SFT 교대).

### Q6. GRPO가 대형 VLA의 RL 미세조정에 적합한 이유를 그룹 상대 어드밴티지 개념과 함께 설명하라.
- GRPO 절차: 그룹 샘플 → 채점 → 평균보다 나은 것 강화.
- 어드밴티지 식 $A_i=(r_i-\mathrm{mean})/\mathrm{std}$ 제시.
- 핵심: **크리틱(가치망) 불필요** → 표준 액터-크리틱의 비싼 가치망을 그룹 평균 베이스라인으로 대체 → 대형 VLA에서 메모리·연산 절약.
- 이전 통찰: 보상이 인간 선호 대신 과제 성공일 뿐 최적화는 동일.

### Q7. VLA RL 방법론의 세 갈래(Online/Offline/Test-time)를 설명하고 각 핵심 trade-off를 논하라.
- Online: 상호작용 후 개선 — 데이터 초과 ↔ 비쌈/위험.
- Offline: 로그 데이터 학습 — 안전·저렴 ↔ 분포 이동.
- Test-time: 실행 시 최선 선택, 재학습 X — 저렴·안전 ↔ 새 기술 학습 불가.

### Q8. Test-time RL의 작동 방식과 한계를 설명하고, 전체 RL 미세조정과 비교해 어떤 상황에 적합한지 논하라.
- 작동: 매 단계 여러 후보 샘플 → 가치 추정으로 순위 → 최상위만 실행, 가중치 불변.
- 장점: 학습 루프 없어 저렴·안전.
- 한계: 기존 정책이 만들 수 있는 후보 중 선택일 뿐 → **진짜 새 기술은 못 배움.**
- 적합 상황: 전체 RL 미세조정이 비용·위험상 부담일 때의 중간 지대.

### Q9. VLA+RL의 주요 열린 과제를 제시하고, 각 과제가 왜 어려운지 도전과제와 연결해 설명하라.
- 보상 설계(게이밍 방지) — 도전 1과 직결.
- Sim-to-real 갭 — 온라인 RL의 본질적 비용/한계.
- 비용·안전 — 실세계 탐색의 위험.
- flow 기반 모델 — 도전 2(행동 표현)와 직결.

### Q10. 코스 전체 흐름(모방 → VLA → DPO/GRPO → VLA+RL)을 하나의 일관된 이야기로 서술하라.
- 관통 아이디어: **"신호로부터 행동을 학습한다"**(보상/시연/선호).
- 아크: 전문가 복사 → 파운데이션 모델로 규모 확장 → RL로 그것을 넘어섬.
- 한 문장 마무리: 모방은 복사를, 강화학습은 개선을 가르쳤다.

## 핵심 용어 빠른 점검 (서술 시 정확히 쓸 것)

- **VLA (Vision-Language-Action)**: 비전·언어를 받아 로봇 행동을 내는 모델. 본질은 규모를 키운 모방.
- **SFT (Supervised Fine-Tuning)**: 시연 모방 단계. 상한 = 시연 품질.
- **GRPO**: 그룹 샘플의 상대 어드밴티지로 정책 업데이트, **크리틱 불필요.**
- **DPO**: 선호 쌍에서 직접 학습.
- **RLOO (REINFORCE Leave-One-Out)**: 나머지 샘플 평균을 베이스라인으로 쓰는 단순 REINFORCE. (RIPT-VLA)
- **Sparse / Verifiable / World-model reward**: 보상 설계 3선택지.
- **Token-based vs Flow/Diffusion-based VLA**: 우도 계산 가능성의 차이 → RL 난이도 차이.
- **Online / Offline / Test-time RL**: 방법론 세 갈래.
- **Distribution shift, Sim-to-real gap, Reward gaming**: 핵심 실패 모드 3종.
