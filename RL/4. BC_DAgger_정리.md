## 0. 큰 그림 한 장 요약 (먼저 외울 서사)

이 두 강의의 전체 논리는 **하나의 인과 사슬**로 이어진다. 서술형 답안의 도입부는 항상 이 사슬에서 시작하면 된다.

1. **탐색 기반 RL은 sparse reward 환경에서 거의 학습이 안 된다** → "정답을 이미 아는데 왜 다시 발견시키는가?"라는 의문.
2. → **Imitation Learning(IL)**: 전문가 시연으로 warm start. 그중 가장 단순한 것이 **Behavior Cloning(BC)**.
3. → BC는 제어 문제를 그냥 지도학습으로 취급한다. 그런데 제어는 **IID 가정을 위반**한다 (현재 행동이 다음 상태를 바꿈).
4. → 이 위반이 **Covariate Shift**를 낳고, 오차가 **$O(T^2\epsilon)$ (2차)**로 폭증한다.
5. → 원인은 "데이터가 학습 전에 고정되어 있어 복구(recovery) 정보가 없다"는 것. **신경망 구조를 바꿔서가 아니라 데이터 분포 자체를 바꿔야** 풀린다.
6. → **DAgger**: 에이전트가 직접 방문한 상태를 전문가가 라벨링해 데이터에 누적(aggregate). 학습 분포가 테스트 분포에 수렴 → **No-regret / FTRL 보장**으로 오차를 **$O(T\epsilon)$ (선형)**으로 복원.
7. → 그러나 DAgger는 **human-in-the-loop** 비용·고차원 상태 평가의 어려움이라는 한계. → 현대 대규모 모델(VLA)은 이 상호작용 루프를 포기하고 **방대한 데이터의 무차별 커버리지**로 대체.

**한 문장 정리**: "BC의 covariate shift($O(T^2\epsilon)$)는 IID 위반에서 비롯되며, DAgger는 데이터 분포를 에이전트 방문 분포에 맞추는 no-regret 온라인 학습으로 이를 $O(T\epsilon)$으로 복원한다. 다만 그 대가는 상호작용 전문가 라벨링 비용이다."

# Part 1. Behavior Cloning (Lecture 13)

## 1.1 제안배경 — 탐색 기반 RL은 왜 실패하는가

**기존 RL(DQN, Policy Gradient, PPO)의 작동 전제**: 에이전트가 무작위 탐색($\epsilon$-greedy, Gaussian noise)으로 보상을 "우연히 발견(stumble upon)"하는 데 의존한다.

이 전제가 무너지는 지점:

- **비효율성(In-efficiency)**: 건초더미에서 바늘 찾기. 수백만 번의 환경 상호작용이 필요.
- **샘플 복잡도(Sample Complexity)**: 탐색 노이즈가 **등방적(isotropic)이고 방향성이 없음(undirected)**. 즉 어디로 가야 할지 모른 채 무작정 흔든다.
- **인간의 노력(Human effort)**: 에이전트를 유도하려면 고통스러운 **Reward Shaping**이 필요.
- **Sparse Reward의 치명성**: 고차원 휴머노이드 보행, 양손 조작(bimanual manipulation)처럼 목표 도달 확률이 거의 0인 과제에서는, 에이전트가 생애의 99.9% 동안 보상 0을 받는다. **따라갈 gradient가 없으니 학습 자체가 시작되지 않는다.**

> **핵심 질문 (강의의 전환점)**: *"이미 과제를 잘 수행하는 방법을 안다면, 왜 에이전트에게 처음부터 다시 발견하라고 강요하는가?"*
> → IL은 전문가 지식으로 **"warm start"**를 제공해 이 문제를 우회한다.

**서술형 포인트**: RL의 한계를 단순히 "느리다"로 쓰지 말 것. ① 탐색 노이즈가 방향성이 없다는 점, ② sparse reward에서 gradient가 0이라 학습이 "시작조차" 안 된다는 점 두 가지를 구분해서 써야 한다.

## 1.2 Imitation Learning의 정식 정의

**정의**: 전문가 정책 $\pi^*$의 시연(demonstration)을 관찰하여, 과제를 수행하는 정책 $\pi_\theta$를 학습하는 과정.

**목적(Objective)**: 명시적 보상함수 $R(s,a)$를 **알거나 관찰하지 않고도** 전문가의 행동을 모방(match)한다. → 이것이 RL과의 결정적 차이.

**입력 — 전문가 데이터셋 $\mathcal{D}$**:

$$\mathcal{D} = \{(s_0, a_0), (s_1, a_1), \dots, (s_T, a_T)\}_{i=1\dots N}$$

데이터 출처: 인간 **teleoperation**, **Motion Capture**, 또는 **legacy 알고리즘 전문가**.

### 데이터 획득 방식: Teleoperation의 양면성

- **장점**: 로봇 데이터를 얻는 가장 널리 쓰이는 방법이며, **깨끗한(clean) 로봇 궤적**을 제공한다. (예: Mobile ALOHA — 저비용 전신 teleoperation으로 양손 모바일 조작 학습)
- **한계 (제안배경으로 자주 출제)**:
  - 비제약(in-the-wild) 환경으로 **확장 불가능(not scalable)**. 어디든 장비를 들고 다니며 유지할 수 없다.
  - **전문가 수준의 조작 기술**이 필요하다.

## 1.3 RL vs IL 비교 (통합 이해 — 서술형 단골)

| 기준 | Reinforcement Learning (RL) | Imitation Learning (IL) |
|------|------|------|
| **주 목표** | 누적 보상 $G_t$ 최대화 | 전문가 행동 $\pi^*$ 모방 |
| **필수 입력** | 보상함수 $R(s,a)$ | 전문가 시연 $\mathcal{D}$ |
| **인간 부담** | 일관된 보상 **설계** | teleop 데이터 **수집/필터링** |
| **수렴 속도** | 느림 (지식 0에서 시작) | 빠름 (지도학습 bootstrap) |
| **일반화** | 시행착오로 최적 행동까지 학습 | 전문가가 "본" 상태로 **제한됨** |

**서술형 핵심**: 이 표의 마지막 두 줄이 핵심 대비점이다.
- IL은 **빠르지만**(수렴), 전문가가 방문한 상태 밖으로 나가면 무력하다(일반화 제한). → 이것이 곧 1.9의 covariate shift 복선.
- RL은 **느리지만** 시행착오를 통해 전문가를 능가하는 최적 행동까지 도달할 수 있다.
- 즉 인간 부담의 성격이 다르다: RL은 "보상 설계"라는 창의적·반복적 부담, IL은 "데이터 수집·필터링"이라는 노동집약적 부담.

## 1.4 Imitation Learning의 분류 체계 (Taxonomy)

네 가지 갈래를 **"보상함수를 다루는 방식"**으로 구분하면 외우기 쉽다.

1. **Behavior Cloning (BC)** — *보상 무시, 직접 모방*
   - 상태→행동을 **오프라인 지도학습**으로 직접 매핑.
   - 가장 단순한 baseline이지만 **compounding execution error(누적 실행 오차)**에 취약.

2. **Interactive Imitation Learning (예: DAgger)** — *상호작용으로 분포 교정*
   - BC의 상호작용 확장. 에이전트가 **자기 궤적을 수집**하고, 온라인 전문가에게서 **실시간 행동 라벨**을 받음.
   - 이 반복 루프가 **covariate shift를 완화**.

3. **Inverse RL (IRL)** — *숨은 보상함수 복원*
   - 전문가 행동을 설명하는 **암묵적 보상함수 $R(s,a)$**를 역으로 추론.
   - 보상을 추론한 뒤 표준 RL로 정책 최적화.

4. **Generative Adversarial Imitation Learning (GAIL)** — *분포 정합*
   - GAN 구조 차용: **정책=생성자(generator)**, **판별자(discriminator)**가 에이전트와 전문가의 **state-action 분포 간 divergence를 최소화**.

**서술형 포인트**: "BC와 IRL의 근본 차이는?" → BC는 보상을 아예 무시하고 행동을 복사하지만, IRL은 보상함수를 복원한 뒤 RL을 돌린다. 따라서 IRL은 전문가를 **능가**할 잠재력이 있고(최적화 여지), BC는 전문가를 **모방**하는 데 그친다.

## 1.5 Behavior Cloning의 본질

BC는 IL의 가장 기본 형태로, **과제의 순차적(sequential) 성질을 무시하고 제어를 지도학습으로 취급**한다.

**가설(Hypothesis)**: 전문가가 사용하는 함수 $f: s \to a$를 학습하면, 자기 관측에 $f$를 그냥 실행해 과제를 수행할 수 있다.

→ 즉 BC = "이미지 분류와 똑같은 지도학습"이라는 관점. 이 **순진한 관점이 곧 약점의 근원**(1.9에서 폭로됨).

## 1.6 BC 손실함수와 최적화

최적화는 이미지 분류처럼 **표준 경사하강(Adam, SGD)**으로 수행.

**연속 제어 (Continuous Control)** — 벡터 간 거리를 최소화하는 MSE:

$$L = \lVert \pi_\theta(s) - a_{\text{expert}} \rVert^2$$

**이산 행동 (Discrete Actions)** — 전문가 선택의 likelihood를 최대화하는 Cross-Entropy:

$$L = -\sum a_{\text{expert}} \log(\pi_\theta(s))$$

**서술형 포인트**: 손실함수 자체보다 "**왜 이미지 분류와 수식이 똑같은가**"가 핵심. BC가 데이터의 IID성을 가정하기 때문이며, 바로 이 가정이 1.9에서 깨진다.

## 1.7 인간 시연으로부터 학습하는 근거 (Rationale)

**핵심 철학 — 생물학적 prior로부터의 bootstrapping**:
- 인간 시연은 지능적 생물학적 제어기가 만든 **고도로 최적화되고 구조화된 궤적**.
- 무질서한 맹목적 탐색으로 안전·기능적 행동을 발견하게 하는 대신, 인간 데이터를 **탐색 공간을 제약하는 prior**로 활용.

**두 가지 활용처**:
1. **BC에서의 효용 — 샘플 효율성**: 관측→전문가 행동을 직접 매핑하면 샘플 소모가 큰 시행착오 단계를 건너뛴다. 복잡한 탐색 문제를 **샘플 효율적인 함수 근사 문제**로 변환.
2. **보상 기반 RL에서의 효용 — sparsity 극복**: 고차원 로봇에서 sparse reward를 우연히 찾을 확률은 0에 가까움. 인간 시연이 **macro 수준 안내자** 역할 — 정책 초기화 또는 dense reference reward 형성에 쓰여 zero-gradient 보상 지형에 갇히지 않게 함.

**통합 가치(Unified Value)**: 하위 알고리즘이 무엇이든(BC로 행동을 직접 복사하든, RL로 궤적 기하를 정렬하든), 인간 시연은 복잡한 로봇 학습을 수학적·실용적으로 **다룰 수 있게 만드는 의미적·물리적 anchor**를 제공한다.

## 1.8 Missing Action Problem (인간 비디오로부터의 학습)

**철학적 전환**: 고비용 로봇 teleoperation에서, **풍부하고 수동적인(passive) 웹 스케일 인간 비디오**로 확장하려는 시도.

**근본 병목**: 인간 비디오는 **관측 시퀀스(상태/이미지)는 주지만, 저수준 제어 명령(행동 라벨)이 전혀 없다**. → "Missing Action Problem".

**세 가지 해결 패러다임** (서술형으로 비교 출제 가능):

1. **Inverse Dynamics Model (IDM)**: 소규모 로봇 데이터로 $(s_t, s_{t+1}) \to a_t$를 예측하는 별도 모델을 학습. 이 IDM으로 수백만 비디오 프레임에 **pseudo-action을 라벨링**한 뒤 오프라인 BC.
2. **Video-Driven Reward + RL**: 인간 비디오를 reference 궤적으로 취급. **Optimal Transport** 등 시각적 정렬 metric으로 dense reward를 정의 → 로봇이 온라인 RL(PPO/SAC)로 유효 행동을 발견. *(← 이 과목 final project)*
3. **Foundation VLA Alignment**: 대규모 사전학습 Vision-Language 모델의 강한 의미적 prior 활용. 인간 비디오가 "무엇을 할지"를 개념적으로 가르치고, 소량의 로봇 데이터가 토큰화된 출력을 정밀한 물리 실행으로 **정렬**.

**서술형 포인트**: 세 패러다임의 차이는 "행동 라벨의 부재를 어떻게 메우는가"이다. ①은 모델로 라벨을 **생성**, ②는 라벨 대신 **보상으로 우회**(행동을 RL이 발견), ③은 대규모 prior로 **의미를 먼저 채우고 소량 데이터로 정렬**.

## 1.9 ⭐핵심⭐ IID 위반과 Covariate Shift

### (1) 숨은 위험 — IID 가정의 위반

지도학습(SL)은 **IID 가정**(데이터가 독립이고 동일 분포)에 의존한다.

- **표준 SL (이미지 분류)**: 인덱스 $n$의 고양이를 오분류해도 인덱스 $n+1$의 이미지는 바뀌지 않는다. **데이터 분포가 정적(static)**.
- **로봇·제어 (Non-IID)**: 잘못된 행동 $a_t$를 취하면 그것이 **직접 다음 상태 $s_{t+1}$를 바꾼다**. 에이전트 자신의 실수가 분포를 이동시킨다.

### (2) 오래된 문제 — ALVINN (Pomerleau, NeurIPS'89)

> *"네트워크는 정확한 주행 예시만 보여줘서는 안 되고, 실수를 저지른 뒤 **어떻게 복구하는지(도로 중앙으로 돌아오는 법)**도 보여줘야 한다."*

→ covariate shift는 **신생 문제가 아니라 1989년부터 알려진 고전 문제**라는 점을 서술하면 좋다. (LeCun'05도 동일 관찰.)

### (3) "로봇은 왜 충돌했는가" — 직관적 그림

- 전문가 시연 궤적(여러 색의 선) 근처에서 에이전트는 작은 오차 $\epsilon$만 낸다.
- 그 작은 오차가 누적되어 에이전트가 **시연이 전혀 없는 영역(No training data)**으로 진입.
- 그 영역에서는 **Error = 1.0** (전문가 라벨이 없으므로 임의 행동) → 더 멀리 이탈 → 충돌(💥).

### (4) Covariate Shift의 정식 정의

**Covariate Shift = 학습 분포와 테스트 분포의 불일치.**

- **학습 시**: 정책은 전문가 방문 상태에서 학습됨, $s \sim d_{\pi^*}$.
- **실행 시**: 정책은 **자기 자신**의 상태를 방문, $s \sim d_{\pi_\theta}$.
- $\pi_\theta \neq \pi^*$이므로, 테스트 중 마주치는 상태가 학습 시 본 상태에서 **drift(표류)**한다.

### (5) 근본 원인 — Feedback Loop

- **표준 SL**: 오차 격리(Error Isolation). 프레임 $t$의 실수가 $t+1$ 입력을 바꾸지 않으므로 시스템이 본질적으로 자가 복구된다.
- **순차 제어 (Non-IID)**: **Action-State Feedback Loop**. $s_{t+1} \sim P(s_{t+1} \mid s_t, a_t)$. 하나의 suboptimal 행동이 로봇을 낯선 상태로 표류시키고, 그 열화된 상태가 다음 입력으로 되먹임 → **악화의 연쇄(cascade)**.

### (6) BC가 유독 취약한 이유

- **물리 노이즈 취약성**: 기계적 backlash, 센서 오차 같은 사소한 실세계 섭동이 로봇을 전문가의 깨끗한 궤적에서 살짝 벗어나게 함.
- **Zero-Shot OOD Panic**: 일단 궤도를 벗어나면 전문가 데이터에 없던 **OOD 상태**에 진입. **복구 라벨이 없으니** 임의 행동을 실행 → **2차 오차 증가($O(T^2)$)**와 시스템 붕괴.

---

## 1.10 ⭐핵심 수식 유도⭐ 오차 증가: $O(T\epsilon)$ vs $O(T^2\epsilon)$

> 서술형 시험에서 **유도 과정을 직접 쓰라**는 문제가 나올 가능성이 높은 부분. 단계별로 외울 것.

### (A) 표준 SL: 선형 오차 $O(T\epsilon)$

- **No Feedback Loop**: 시점 $t$의 예측이 $t+1$의 입력에 영향을 주지 않음.
- **Error Isolation**: 각 단계 실수 확률 $\epsilon$이 완전히 격리됨. $t+1$ 입력은 여전히 참 분포에서 깨끗하게 추출.
- 오차가 누적·전파되지 않으므로 단순 합산:

$$\text{Total Error}_{SL} \le \sum_{t=1}^{T} \epsilon = T\epsilon = O(T\epsilon)$$

### (B) Behavior Cloning: 2차 오차 $O(T^2\epsilon)$

**Worst-case 상한 분석**: 일단 OOD 상태에 강제 진입하면 그 상태의 전문가 라벨이 없으므로, **최악의 시나리오 = 그 에피소드의 남은 모든 단계에서 확률 1로 실패**한다고 가정한다.

**유도**:
1. 에이전트가 **첫 실수를 시점 $t$에 저지름** (확률 상한 $\epsilon$).
2. 그러면 남은 $T-t$ 단계 동안 계속 실패.
3. 전체 기대 오차 = (시점 $t$에 첫 실수할 확률) × (남은 실패 단계 수)를 전 구간에 대해 적분:

$$\text{Total Error}_{BC} \le \sum_{t=1}^{T} \epsilon \cdot (T - t)$$

4. 산술급수로 전개:

$$\sum_{t=1}^{T} \epsilon(T-t) = \epsilon\left[(T-1) + (T-2) + \dots + 1 + 0\right]$$

5. 합 공식 $\frac{n(n+1)}{2}$에 $n = T-1$ 대입:

$$\text{Total Error}_{BC} \le \epsilon \cdot \frac{(T-1)T}{2} = \frac{\epsilon}{2}(T^2 - T) = O(T^2\epsilon)$$

**서술형 핵심 비교**:
- SL은 오차가 **격리**되어 선형 $O(T\epsilon)$.
- BC는 첫 실수 이후 **남은 전 구간이 오염**되므로, "실수 확률 × 잔여 구간"을 더하면 **2차 $O(T^2\epsilon)$**.
- 직관: SL의 실수는 "점"이고, BC의 실수는 "그 시점부터 끝까지 이어지는 꼬리"다. 이 꼬리들의 총합이 삼각형(=2차) 면적이 된다.

## 1.11 최신 SOTA VLA와 BC — 역설적 현실

- 더 발전된 IL 알고리즘(DAgger 등)이 있음에도, **최신 SOTA VLA는 그냥 BC를 쓴다**. 시간 소모적인 rollout(상호작용 수집)이 필요 없기 때문.
- "그럼 covariate shift는?" → **"그냥 대규모 데이터를 쓴다"**는 접근. 방대한 데이터가 OOD 상태까지 어느 정도 커버.
- 그러나 **일반화 실패(generalization failure)는 여전히 미해결 이슈**.
- (참고: OpenVLA는 행동을 토큰으로 이산화한 뒤, Llama tokenizer의 가장 덜 쓰이는 256개 토큰을 행동 토큰으로 덮어쓰고, **표준 next-token prediction**으로 학습 — 즉 본질적으로 cross-entropy 기반 BC.)

**서술형 포인트**: "이론적으로 우월한 DAgger 대신 왜 산업계는 BC를 쓰는가?"는 좋은 통합 문제. 답: **상호작용 비용 vs 데이터 규모의 트레이드오프**. DAgger의 no-regret 보장은 매력적이지만 online expert rollout 비용이 막대하고, 대규모 모델 시대에는 "데이터를 늘려 분포를 넓히는" 무차별 커버리지가 공학적으로 더 현실적이기 때문. (단, 일반화 실패는 남음 → Part 2.14로 연결)

# Part 2. DAgger (Lecture 14)

## 2.1 제안배경 — Covariate Shift 위기 재정리

DAgger 강의는 BC의 실패를 다시 못 박으며 시작한다:
- 오프라인 BC는 **극도로 취약(exceptionally fragile)**.
- 학습 분포(전문가 궤적)와 테스트 분포(에이전트 궤적)의 불일치가 **지속적 drift**를 일으킴.
- 데이터셋에 없는 상태에 진입하면 **임의 행동** 출력 → 나쁜 행동이 더 나쁜 상태를 낳는 **feedback loop**.

**오늘의 목표(명시적 선언)**: *"이 2차 의존성($O(T^2\epsilon)$)을 수학적으로 깨뜨려 선형 bound($O(T\epsilon)$)로 복원하는 것."*

---

## 2.2 ⭐중요 통찰⭐ 오프라인 학습의 근본적 결함

BC는 **수동적 데이터 수집 단계 → 격리된 학습 단계**를 가정한다.

- 데이터셋이 에이전트가 과제를 시도하기도 **전에 완전히 고정**된다.
- 결과적으로 데이터셋에는 "학습된 에이전트가 저지르기 쉬운 **특정 실수로부터 어떻게 복구하는가**"에 대한 정보가 **0**이다.

> **결론 (서술형 단골 명제)**: *"Covariate Shift는 신경망 **구조(architecture)**를 바꿔서는 풀 수 없다. **데이터 분포 자체**를 바꿔야 한다."*

이 한 문장이 DAgger 존재 이유의 전부다. 답안에서 반드시 인용할 것.

---

## 2.3 Interactive Imitation Learning이라는 패러다임

- 분포 불일치를 고치려면 **에이전트가 실제로 방문하는 상태를 커버하는 데이터**가 필요.
- Interactive IL은 **데이터 수집과 정책 학습을 교차(interleave)**시키는 패러다임.
- 전문가가 학습 과정 내내 관여 — **수동적 시연자가 아니라 능동적 oracle(신탁)**.
- 덕분에 에이전트가 **지금 실패하고 있는 바로 그 상태**에 대해 구체적으로 지도를 요청할 수 있다.

---

## 2.4 DAgger (Dataset Aggregation) 개요

- **제안**: Stéphane Ross, Geoffrey Gordon, Drew Bagnell (AISTATS 2011).
- **정의**: **no-regret 온라인 학습**을 달성하도록 결정론적(deterministic) 정책을 학습하는 **반복(iterative) 알고리즘**.
- **핵심 직관**: 에이전트가 자신의 결함 있는 정책을 실행하게 두고 → 방문한 상태를 기록 → 그 **특정 상태들에 대해 전문가가 라벨**을 달게 한다.
- 새 라벨을 기존 데이터와 **누적(aggregate)**함으로써, 학습 분포가 **반복적으로 테스트 분포에 접근**한다.

### DAgger 알고리즘 (의사코드 — 통째로 외울 것)

```
Initialize  D ← ∅
Initialize  π̂₁ ← Π 내 임의 정책
for i = 1 to N do
    π_i = β_i·π* + (1 − β_i)·π̂_i          # (Step 2) mixing policy
    Sample T-step trajectories using π_i    # (Step 2) rollout
    D_i = {(s, π*(s))}  visited by π_i       # (Step 3) 전문가 라벨링
    D ← D ∪ D_i                              # (Step 4) aggregate
    Train classifier π̂_{i+1} on D            # (Step 5) 재학습
end for
Return  best π̂_i on validation
```

## 2.5 ⭐핵심 이론⭐ No-Regret Learning

DAgger의 선형 bound 복원은 전적으로 이 개념 위에 서 있다.

**Regret(후회)의 정의**: 에이전트가 시간에 걸쳐 실제로 저지른 총 실수와, **처음부터 단 하나의 최선 전략을 골랐다면** 저질렀을 최소 실수의 **차이**. 즉 "돌이켜봤을 때 다르게 행동했더라면" 하는 아쉬움의 크기.

**No-Regret Learning의 정의**: 데이터와 반복이 늘수록 **평균 regret이 0으로 수렴**하는 순차 학습 시스템. 수학적으로, 총 regret이 총 단계 수 $N$보다 훨씬 느리게 증가:

$$\lim_{N\to\infty} \frac{\text{Regret}}{N} \le 0$$

**DAgger에 중요한 이유**:
- DAgger는 데이터 수집을 **단계별 온라인 게임**으로 취급. 각 반복에서 에이전트는 **누적된 모든 과거 데이터로 학습**해 다음 정책을 고른다.
- 이 메커니즘이 No-Regret 틀에 정확히 들어맞아, 에이전트의 전체 전략이 결국 **사후 최적 전문가 baseline만큼 좋아짐**을 보장.
- → 이 No-Regret 보장이 **2차 오차 증가를 깨고 선형 bound로 복원하는 정확한 수학적 방패(shield)**.

---

## 2.6 DAgger 5단계 상세

### Step 1 — Initial Training (초기 학습)
- 표준 BC와 **정확히 동일하게** 시작.
- 전문가가 깨끗한 오프라인 시연 데이터셋 제공.
- 첫 정책을 이 초기 데이터에 대한 지도학습으로 훈련.
- 이 단계에서는 **최적 상태만 봤으므로 drift에 매우 취약**.

### Step 2 — Agent Rollout (에이전트 전개)
- 불완전한 $\hat\pi_i$를 **완전 자율로 실행하지 않고**, **혼합 제어 정책 $\pi_i$**로 $T$-step 궤적을 수집:

$$a_{\text{rollout}} = \beta_i\,\pi^*(s) + (1-\beta_i)\,\hat\pi_i(s)$$

- 초기 반복엔 $\beta_i \approx 1$ (전문가의 안전 가이드를 크게 의존), 점차 $\beta_i \to 0$ (에이전트에게 자율권 이양).
- 이 blending이 **안전 버퍼(safety buffer)** 역할 — 즉각적 시스템 붕괴 없이 에이전트 자율성 증가로 인한 **정확한 궤적 drift를 기록**.
- 환경은 이 hybrid 루프에서 방문한 **모든 상태 $s$를 로깅**(다음 전문가 교정의 원천 입력).

### Step 3 — Expert Labeling (전문가 라벨링) ★DAgger의 정의적 메커니즘★
- oracle이 에이전트 rollout에서 기록된 상태들을 검토.
- **모든 상태에 대해** 전문가가 "취했어야 할 최적 행동"을 제공.
- → **(에이전트 상태, 전문가 행동)** 쌍의 새 데이터셋 생성.
- 본질적으로 전문가가 에이전트에게 **자기 고유의 실수로부터 복구하는 법**을 가르침.

### Step 4 & 5 — Aggregation & Retraining (누적과 재학습)
- 새 데이터셋을 **이전 모든 데이터셋과 연결(concatenate)**.
- 데이터셋이 계속 성장 — 초기 전문가 경로와 이후 모든 복구 경로의 **기억을 유지**.
- 이 포괄적 누적 데이터에 대해 **완전히 새 정책을 처음부터(또는 fine-tune) 학습**. → **단순 지도학습!**
- 과정 반복 → **상태 분포가 안정화(stabilize)**.

**서술형 포인트**: Step 3이 "정의적 메커니즘"인 이유를 명시할 것. 핵심은 라벨이 붙는 상태가 **전문가가 방문한 상태가 아니라 에이전트가 방문한 상태**라는 점. 이것이 학습 분포를 테스트 분포에 맞추는 유일한 장치다.

---

## 2.7 전문가 정책 $\pi^*$은 어디서 오는가 (실용적 출처)

서술형으로 "DAgger의 oracle을 실제로 어떻게 구현하나?"가 나올 수 있다.

1. **Legacy Algorithmic Oracles (알고리즘 신탁)**: 시뮬레이터 백엔드에 실시간 기하 planner(MPC, A\*, RRT)를 구현해, 임의의 커스텀 상태에서 **수학적 ground truth**를 계산.
2. **Privileged RL Experts (특권적 RL 전문가)**: 노이즈 없는 oracle 시뮬레이터 변수(예: 정확한 물체 좌표)를 입력받아 깨끗한 반응 행동을 내는 **사전학습 고성능 PPO 정책**.
   - 이런 특권 데이터는 보통 **학습 시에만** 가용, 테스트 시엔 불가용.
   - **하지만 상관없다** — 전문가 정책 $\pi^*$는 **학습 시에만** 쓰기 때문. *(중요한 개념: teacher는 테스트에 등장하지 않는다.)*
3. **Active Human Teleoperation (능동적 인간 원격조작)**: rollout 중 VR/조이스틱으로 교정 제어 라벨을 **수동 주입**하는 인간 운영자를 학습 루프에 상주.

---

## 2.8 DAgger 예시 — Teacher(특권) vs Student(현실)

이 예시는 "DAgger가 실제로 무엇을 해결하는가"를 가장 잘 보여주는 시나리오.

**설정**:
- **Expert/Teacher ($\pi^*$)**: 고품질 teleop 상태, 정확한 3D 물체 좌표, 원시 시뮬레이터 상태 벡터 같은 **"특권(privileged)" 데이터** 접근.
- **Student ($\hat\pi$)**: 현실적 배포 제약 하에서 **단일 RGB 이미지** 또는 저차원 기하 특징만 받음.

**표준 BC가 여기서 실패하는 이유**: Student를 완전 오프라인으로 학습하면 즉시 분포 drift. Student는 전체 teleop 상태를 처리할 수 없으므로, 사소한 시각적 occlusion이나 센서 노이즈가 Student를 **학습 데이터가 없는 OOD 시각 프레임**으로 밀어 넣음.

**DAgger의 해결**:
- **Step 1**: Student가 단일 이미지 입력으로 rollout → 필연적으로 궤도 이탈.
- **Step 2**: 그 지저분한(drifted) 프레임들을 **특권 Expert($\pi^*$)**에게 전달.
- **Step 3**: Expert가 특권 teleop/시뮬레이터 백엔드 데이터로 그 **특정 시각 각도에서의 정확한 교정 행동**을 계산.
- **결과**: Student가 **단일 이미지 입력만으로 복구 경로를 학습** → 테스트 시 값비싼 teleop 셋업 없이 covariate shift 장벽을 돌파.

**서술형 핵심**: 이 예시의 묘미는 **관측 비대칭(observation asymmetry)**이다. Teacher는 특권 정보로 라벨을 만들지만, Student는 빈약한 관측으로 그 라벨을 모방하도록 학습된다. 따라서 비싼 센서는 **학습 시에만** 필요하고 배포 시엔 불필요 — 이것이 DAgger의 실용적 가치.

---

## 2.9 ⭐핵심⭐ IID 위반을 깨는 메커니즘

DAgger는 **정의에 의해(by definition)** 학습 분포가 테스트 분포와 일치하도록 강제한다.

- 반복이 진행될수록 누적 데이터 $\mathcal{D}$는 **에이전트가 자연히 방문하는 상태**들로 지배(dominated)된다.
- 따라서 $\mathcal{D}$에서 상태를 샘플링해 loss를 계산하는 것이, **에이전트의 기대 상태 분포에서 샘플링하는 것과 수학적으로 등가**가 된다.
- → 명시적 보상함수 없이도 **Covariate Shift를 우아하게 중화(neutralize)**.

**서술형 포인트**: "DAgger가 covariate shift를 푸는 본질"을 한 줄로: *학습 시 $s\sim d_{\pi^*}$였던 것을, 반복을 통해 $s\sim d_{\pi_\theta}$로 강제로 이동시켜 학습=테스트 분포로 만든다.* (1.9의 정의를 정확히 뒤집는 구조임을 보일 것.)

---

## 2.10 ⭐이론적 핵심⭐ Online Learning으로의 환원 & FTRL

### Online Learning으로의 환원
- DAgger는 IL을 **단일 지도학습 문제가 아니라 온라인 학습 시퀀스**로 정식화.
- 각 반복에서 알고리즘이 정책을 고르면, 환경이 그 정책의 상태 분포 기반 **loss 함수를 드러낸다**.
- 목표: **사후(in hindsight) 최선 정책에 대해 no-regret**을 달성하는 정책 시퀀스를 찾기.

### Loss 함수 정식화
정책 $\pi$가 반복 $i$에서 생성하는 상태 분포 $d_{\pi_i}$ 하의 기대 loss:

$$L_i(\pi) = \mathbb{E}_{s\sim d_{\pi_i}}\,[\,\ell(s, \pi)\,]$$

- 핵심: 어떤 정책의 loss는 **$\pi_i$ 실행 중 마주친 상태**에 근거해서만 평가된다.
- DAgger는 모든 과거 반복에 걸쳐 이 **누적 loss를 최소화**하는 정책 시퀀스를 찾음으로써 작동.

### DAgger = FTRL (Follow-The-Regularized-Leader)

**FTRL이란?**: 온라인 학습 최적화의 기초 알고리즘. **관측된 loss 함수의 전체 역사를 보고** 다음 파라미터를 고른다.
- 현재 단계만 보는 국소적 미세조정 대신, **모든 과거 반복의 총 누적 loss를 최소화하는 절대 최선 정책("leader")**을 선택.
- 노이즈 있는 온라인 데이터로 정책이 너무 급격히 흔들리지 않도록 **정규화 항 $R(\pi)$**로 최적화 경로를 안정화.

$$\hat\pi_{i+1} = \arg\min_{\pi\in\Pi}\ \sum_{j=1}^{i} L_j(\pi) + R(\pi)$$

**DAgger와의 직접 대응**: 역사적 데이터를 합쳐 새 분류기를 처음부터 재학습하는 것($\mathcal{D}\leftarrow\mathcal{D}\cup\mathcal{D}_i$)이 **정확히 FTRL처럼 동작**한다. 성장하는 데이터셋이 곧 누적 loss 합 $\sum L_j(\pi)$를 표현하기 때문.

**No-Regret 보장 (최종 부등식)**: FTRL이 검증된 no-regret 알고리즘이므로, 반복 $N$이 커지면 평균 regret이 0으로 수렴 — 평균 경험 loss가 사후 최선 고정 baseline에 수렴:

$$\frac{1}{N}\sum_{i=1}^{N} L_i(\pi_i)\ -\ \min_{\pi}\frac{1}{N}\sum_{i=1}^{N} L_i(\pi)\ \le\ o(1)$$

**서술형 포인트**: "DAgger가 왜 no-regret인가?"를 묻는다면, **"데이터 누적 + 처음부터 재학습 = FTRL"**이라는 대응을 반드시 명시. 신경망을 재학습하는 단순 행위가 사실은 누적 loss를 푸는 FTRL이라는 점이 이론적 핵심.

---

## 2.11 Linear Error Bound 복원 — 결론

- No-regret 온라인 최적화의 수학적 성질을 활용해, DAgger는 훨씬 더 tight하고 안전한 상한을 보장.
- 기저 지도 분류기가 단계당 오차 $\epsilon$을 달성하면, 총 기대 과제 loss는 실행 horizon $T$에 **선형**으로 scale:

$$\text{Total Expected Error} \le O(T\epsilon)$$

- 이는 순수 오프라인 BC의 위험한 **2차 증가 $O(T^2\epsilon)$**에 대한 **주요 이론적 개선**.

**한 줄 핵심**: $O(T^2\epsilon) \xrightarrow{\text{No-Regret/FTRL}} O(T\epsilon)$. (이 화살표가 DAgger 강의 전체의 결론.)

---

## 2.12 Mixing Parameter $\beta_i$ — 안전과 탐색의 손잡이

**왜 필요한가**: 완전히 미학습된 정책을 초기 반복에 완전 자율로 실행하면 매우 비효율적이거나 실세계에서 **물리적으로 위험**.

**정의**: 실제 데이터 수집 정책은 전문가와 현재 모델의 선형 결합:

$$\pi_i = \beta_i\,\pi^* + (1-\beta_i)\,\hat\pi_i$$

- 행동 수준에서 연속 제어 벡터를 선형 평균하거나, 매 시점 두 제어기 사이를 **확률적으로 전환**.
- $\beta_i \to 0$으로 decay시켜 전문가 전적 의존 → 에이전트 완전 자율로 **부드럽게 전환**.

### Exploration vs Safety 트레이드오프 (서술형 단골)
$\beta_i$는 본질적으로 **최적 경로 주변의 탐색 반경**을 제어한다.

- **너무 높게 유지하면**: 에이전트가 실패 상태를 결코 방문하지 않음 → **복구 데이터가 수집되지 않음**. (DAgger의 목적 자체가 무력화)
- **너무 빨리 떨어뜨리면**: 에이전트가 **복구 불가능한 상태(unrecoverable)**로 너무 멀리 표류 → 무관한(irrelevant) 데이터 생성.
- → 이 스케줄 튜닝이 **안정적 경험 성능에 결정적**.

**서술형 핵심**: $\beta$ 스케줄은 "복구 데이터를 얻을 만큼은 실패하게 두되, 회복 불가능할 만큼은 이탈하지 않게" 하는 균형. 너무 높음=실패 미수집, 너무 낮음=무의미한 실패 수집.

---

## 2.13 DAgger의 한계 (문제점 — 확장 방향 서술의 출발점)

### (1) Human-in-the-Loop 병목
- 수학적으로 건전하지만, **능동적 oracle 라벨링 요구**가 가장 큰 공학적 약점.
- 복잡한 연속 제어에서, 수천 개 상태마다 실행을 멈추고 인간이 정밀한 차원 행동 벡터를 제공하는 것은 **비현실적(prohibitive)**.
- **인간 피로(fatigue)**가 noisy·불일치 라벨을 낳아, DAgger가 의존하는 바로 그 지도학습 신호를 **열화**시킴.

### (2) State-Space Complexity (상태 공간 복잡도)
- 고차원 시각 공간에서, 임의의 실패 상태를 평가하고 최적 복구 행동을 예측하는 것은 **전문가에게도 어렵다**.
- 인간은 **연속적·유려한 과제 실행엔 능하지만, 이산적·프레임 단위 상태 평가엔 서툴다**.
- 이 **인지적 불일치(cognitive mismatch)**가 DAgger의 효용을 **알고리즘 oracle이 가용한 환경 밖에서는 크게 제한**.

**서술형 핵심**: DAgger의 한계는 "이론이 틀려서"가 아니라 "**전문가 oracle을 매 상태마다 호출하는 비용**" 때문임을 분명히 할 것. 특히 (2)는 미묘함: 인간이 "직접 운전(연속 실행)"은 잘해도 "이 프레임에서 무슨 행동이 옳았나(이산 평가)"는 잘 못한다는 인지 비대칭이 핵심.

---

## 2.14 현대 패러다임으로의 전환 (확장 방향)

- 현대 AI의 거대한 스케일링 요구로 인해, **상호작용 라벨링은 종종 완전히 포기됨**.
- 분야는 **무차별 커버리지(brute-force coverage)**로 이동: 노이즈와 복구를 본질적으로 포함하는 **방대한 데이터셋**을 수집해, 상호작용 루프의 필요를 **중화**.

**서술형 핵심 — 통합 결론**: BC(1.11) ↔ DAgger(2.14)의 현대적 재회. DAgger는 "데이터 분포를 능동적으로 교정"하지만, VLA 시대는 "데이터를 압도적으로 늘려 분포를 수동적으로 커버"한다. 이론적으로 DAgger가 우월하나, **확장성(scalability) 앞에서 산업계는 데이터 규모를 택했다.** 단, 이는 covariate shift를 "근본적으로 푼" 것이 아니라 "데이터로 가린" 것이며, **일반화 실패는 여전히 열린 문제**다.

# Part 3. 통합 정리 & 예상 서술형 문제

## 3.1 핵심 개념 5문장 압축
1. **BC**: 제어를 IID 지도학습으로 취급 → 단순하지만 IID 위반으로 covariate shift 발생.
2. **Covariate Shift**: 학습 분포($d_{\pi^*}$) ≠ 테스트 분포($d_{\pi_\theta}$). feedback loop 때문에 작은 오차가 OOD로 누적.
3. **오차 bound**: SL은 오차 격리로 $O(T\epsilon)$, BC는 첫 실수 이후 잔여 구간 오염으로 $O(T^2\epsilon)$.
4. **DAgger**: 에이전트 방문 상태를 전문가가 라벨링·누적 → 학습=테스트 분포. 이는 FTRL(no-regret)과 등가 → $O(T\epsilon)$ 복원.
5. **한계·전환**: oracle 라벨링 비용·인지 불일치 → 현대 VLA는 상호작용 대신 대규모 데이터 커버리지로 대체.

## 3.2 핵심 수식 모음 (외울 것)

| 항목 | 수식 |
|------|------|
| 전문가 데이터셋 | $\mathcal{D}=\{(s_t,a_t)\}_{i=1\dots N}$ |
| BC 연속 손실 | $L=\lVert\pi_\theta(s)-a_{\text{expert}}\rVert^2$ |
| BC 이산 손실 | $L=-\sum a_{\text{expert}}\log(\pi_\theta(s))$ |
| SL 오차 | $\le \sum_{t=1}^{T}\epsilon = O(T\epsilon)$ |
| BC 오차 | $\le \sum_{t=1}^{T}\epsilon(T-t)=\epsilon\frac{(T-1)T}{2}=O(T^2\epsilon)$ |
| 상태 전이 | $s_{t+1}\sim P(s_{t+1}\mid s_t,a_t)$ |
| DAgger mixing | $\pi_i=\beta_i\pi^*+(1-\beta_i)\hat\pi_i$ |
| DAgger loss | $L_i(\pi)=\mathbb{E}_{s\sim d_{\pi_i}}[\ell(s,\pi)]$ |
| FTRL | $\hat\pi_{i+1}=\arg\min_{\pi\in\Pi}\sum_{j=1}^{i}L_j(\pi)+R(\pi)$ |
| No-regret | $\lim_{N\to\infty}\frac{\text{Regret}}{N}\le 0$ |
| DAgger 보장 | $\frac{1}{N}\sum L_i(\pi_i)-\min_\pi\frac{1}{N}\sum L_i(\pi)\le o(1)$ |
| DAgger 오차 | $\le O(T\epsilon)$ |

## 3.3 예상 서술형 문제 + 답안 개요

**Q1. Behavior Cloning의 오차가 표준 지도학습과 달리 $O(T^2\epsilon)$로 증가하는 이유를 수식 유도와 함께 설명하라.**
→ 개요: IID 위반(feedback loop) 설명 → SL은 오차 격리로 $\sum\epsilon=O(T\epsilon)$ → BC는 첫 실수(확률 $\epsilon$) 후 잔여 $T-t$ 단계 worst-case 실패 → $\sum\epsilon(T-t)=\epsilon(T-1)T/2=O(T^2\epsilon)$ → 직관(SL=점 오차, BC=꼬리 오차의 삼각형 면적).

**Q2. Covariate shift란 무엇이며, 왜 신경망 구조 개선만으로는 해결할 수 없는지 논하라.**
→ 개요: 정의($d_{\pi^*}$ vs $d_{\pi_\theta}$) → 원인은 오프라인 데이터가 학습 전 고정되어 복구 정보 0 → 따라서 architecture가 아니라 **데이터 분포 자체**를 바꿔야 함 → DAgger로 연결.

**Q3. DAgger 알고리즘의 5단계를 설명하고, 어느 단계가 covariate shift 해결의 핵심인지 근거와 함께 밝혀라.**
→ 개요: Step1~5 서술 → **Step 3(Expert Labeling)**이 핵심, 이유는 라벨이 붙는 상태가 전문가 방문 상태가 아닌 **에이전트 방문 상태**이기 때문 → 2.9의 분포 일치 메커니즘.

**Q4. DAgger가 $O(T^2\epsilon)$를 $O(T\epsilon)$로 복원하는 이론적 근거(no-regret, FTRL)를 설명하라.**
→ 개요: 온라인 학습 환원 → loss $L_i(\pi)=\mathbb{E}_{s\sim d_{\pi_i}}[\ell]$ → 데이터 누적·재학습 = FTRL($\arg\min\sum L_j+R$) → FTRL은 no-regret($\text{Regret}/N\to 0$) → 평균 loss가 사후 최선 baseline 수렴 → 선형 bound.

**Q5. Mixing parameter $\beta_i$의 역할과 스케줄링 트레이드오프를 설명하라.**
→ 개요: $\pi_i=\beta_i\pi^*+(1-\beta_i)\hat\pi_i$ → 초기 안전 버퍼($\beta\approx1$), 점차 자율($\beta\to0$) → 너무 높음=복구 데이터 미수집, 너무 낮음=복구 불가 상태로 이탈해 무의미 데이터 → 탐색 반경 제어로서의 의미.

**Q6. RL, BC, DAgger를 인간 부담·수렴·일반화·이론적 보장 관점에서 비교하라.**
→ 개요: RL(보상 설계 부담, 느림, 최적 일반화) vs BC(데이터 수집 부담, 빠름, 전문가 상태 제한, $O(T^2\epsilon)$) vs DAgger(상호작용 oracle 부담, $O(T\epsilon)$ 보장, 분포 일치). 트레이드오프로 마무리.

**Q7. 인간 비디오로부터 학습할 때의 Missing Action Problem과 세 가지 해결 패러다임을 비교하라.**
→ 개요: 병목(관측은 있으나 행동 라벨 부재) → IDM(라벨 생성) / Video reward+RL(보상으로 우회) / Foundation VLA(prior로 의미 채우고 소량 데이터로 정렬) → 각각의 가정·비용 비교.

**Q8. DAgger의 한계를 두 가지 들고, 현대 대규모 모델(VLA)이 이를 어떻게 우회하는지 논하라.**
→ 개요: ① human-in-the-loop 비용·라벨 노이즈 ② 고차원 상태의 이산 평가 인지 불일치 → VLA는 상호작용 포기·무차별 데이터 커버리지로 대체 → 그러나 일반화 실패는 미해결(이론적 우월 vs 확장성의 트레이드오프).

## 3.4 자주 헷갈리는 포인트 점검
- **BC vs DAgger의 데이터 출처**: BC는 전문가 방문 상태 $(s,\pi^*(s)),\ s\sim d_{\pi^*}$. DAgger는 **에이전트 방문 상태** $s\sim d_{\pi_i}$에 전문가가 라벨. ← 가장 중요한 구분.
- **$\pi^*$는 테스트에 등장하지 않는다**: 특권 정보 기반 전문가는 학습 시에만 사용. Student는 빈약한 관측으로 그 라벨을 모방. (2.7, 2.8)
- **IRL vs BC**: IRL은 보상함수를 복원 후 RL → 전문가 능가 가능. BC는 행동 직접 복사 → 모방에 그침.
- **No-regret의 의미**: "총 regret이 0"이 아니라 "**평균** regret이 0으로 수렴"($\text{Regret}/N\to0$). 총 regret은 $N$보다 느리게 증가.
- **$O(T^2\epsilon)$의 worst-case 가정**: 첫 실수 후 남은 전 구간을 확률 1로 실패한다고 가정한 **상한(upper bound)**. 실제론 더 낮을 수 있으나 최악 분석에서 2차가 나온다.
