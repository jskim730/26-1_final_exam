### 주제: GAE (Generalized Advantage Estimation) & TRPO (Trust Region Policy Optimization)

> **이 자료의 사용법** — 시험이 모두 서술형이므로, 각 개념을 **(1) 왜 등장했는가(제안 배경) → (2) 어떻게 작동하는가(원리·수식) → (3) 무엇이 문제인가(한계) → (4) 어디로 발전하는가(확장)** 의 4단 구조로 설명할 수 있도록 정리했습니다. 마지막 섹션에 예상 서술형 문제와 답안 골격을 넣었습니다.

## 0. 전체 그림: 이 강의가 풀려는 문제

정책 경사(Policy Gradient) 계열 알고리즘은 결국 두 개의 독립적인 난제를 동시에 풀어야 합니다.

1. **신호의 질(Signal Quality) 문제** — "이 행동이 평균보다 얼마나 좋았는가"를 나타내는 **Advantage 추정치 $\hat{A}$** 가 편향(bias)과 분산(variance) 사이에서 흔들립니다. → **GAE가 담당**
2. **업데이트의 안전성(Update Stability) 문제** — 추정한 경사 방향으로 **얼마나 큰 보폭(step size)** 으로 가야 정책이 망가지지 않는가. → **TRPO가 담당**

이 둘은 PPO를 떠받치는 두 기둥입니다. GAE는 "무엇을 향해 갈지(방향의 품질)", TRPO는 "얼마나 멀리 갈지(보폭의 안전)"를 책임집니다. 서술형에서는 **이 두 문제가 별개의 축이라는 점**을 먼저 짚고 들어가면 좋은 인상을 줍니다.

또한 두 기법 모두 공통적으로 **중요도 샘플링(Importance Sampling, IS)** 위에서 동작합니다. IS는 옛 정책 $\pi_{old}$로 모은 데이터를 현재 정책 $\pi_\theta$ 학습에 재사용(off-policy)하게 해 주는 장치인데, 이때 곱해지는 비율 $\frac{\pi_\theta(a|s)}{\pi_{old}(a|s)}$ 이 노이즈를 증폭시키므로, **곱해지는 상대인 Advantage가 깨끗해야(GAE) IS가 안정적으로 작동**합니다.

## 1. GAE (Generalized Advantage Estimation)

### 1-1. 제안 배경 — Bias–Variance 딜레마

Advantage를 추정하는 방법은 N-step 추정량의 스펙트럼 위에 놓여 있습니다.

$$\hat{A}_t^{(1)} = r_t + \gamma V(s_{t+1}) - V(s_t)$$
$$\hat{A}_t^{(2)} = r_t + \gamma r_{t+1} + \gamma^2 V(s_{t+2}) - V(s_t)$$
$$\hat{A}_t^{(\infty)} = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots - V(s_t)$$

- **1-step (TD, $k=1$)**: 실제 보상은 한 스텝만 쓰고 나머지는 학습된 가치함수 $V$에 의존 → **분산은 낮지만**, $V$가 부정확하면 **편향이 큼**.
- **무한 step (MC, $k=\infty$)**: 궤적 전체의 실제 보상만 사용 → **편향은 없지만(unbiased)**, 환경의 확률적 노이즈가 누적되어 **분산이 폭발**.

핵심 문제의식: **MC의 무편향성과 TD의 저분산을 동시에 갖고 싶다.** 게다가 이 분산 큰 추정치에 IS 비율까지 곱하면 분산이 한 번 더 증폭되므로(강의 6쪽), 단일 N-step 추정으로는 안정적인 off-policy 학습이 불가능합니다. GAE는 "하나의 $k$를 고르지 말고 **모든 $k$를 지수가중 평균하자**"는 발상입니다.

### 1-2. 핵심 도구 — TD-error와 텔레스코핑(telescoping)

TD-error를 정의하면:

$$\delta_t^V = r_t + \gamma V(s_{t+1}) - V(s_t)$$

이때 N-step advantage는 **TD-error들의 할인 합**으로 깔끔하게 정리됩니다(강의 8쪽).

$$\hat{A}_t^{(k)} = \sum_{l=0}^{k-1} \gamma^l \delta_{t+l}^V = \sum_{l=0}^{k-1}\gamma^l r_{t+l} + \gamma^k V(s_{t+k}) - V(s_t)$$

> **이게 왜 중요한가(서술 포인트)**: 중간의 $V(s_{t+1}), V(s_{t+2}) \dots$ 항들이 서로 상쇄(telescope)되어, $k$가 달라도 동일한 TD-error 블록을 쌓는 형태로 표현됩니다. 덕분에 "모든 $k$의 평균"을 닫힌 형태로 계산할 수 있게 됩니다.

### 1-3. GAE 정의와 유도

GAE는 모든 $k$-step 추정량을 $\lambda$로 지수가중 평균한 것입니다.

$$\hat{A}_t^{GAE(\gamma,\lambda)} = (1-\lambda)\left(\hat{A}_t^{(1)} + \lambda \hat{A}_t^{(2)} + \lambda^2 \hat{A}_t^{(3)} + \cdots\right)$$

이를 TD-error로 풀어 쓰고 등비급수 합($\sum_{k=0}^\infty \lambda^k = \frac{1}{1-\lambda}$)을 정리하면, 매우 단순한 최종 형태로 수렴합니다(강의 12쪽):

$$\boxed{\;\hat{A}_t^{GAE(\gamma,\lambda)} = \sum_{l=0}^{\infty}(\gamma\lambda)^l\,\delta_{t+l}^V\;}$$

> **유도 흐름 요약(서술용)**: ① 각 $\hat{A}^{(k)}$를 TD-error 합으로 치환 → ② $\delta_t^V$, $\delta_{t+1}^V$, $\delta_{t+2}^V$ 별로 계수를 묶음 → ③ 각 묶음이 등비급수가 되어 $\frac{1}{1-\lambda}$ 가 나오고 앞의 $(1-\lambda)$와 상쇄 → ④ 결과적으로 TD-error에 $(\gamma\lambda)^l$ 가중치가 붙은 깔끔한 합이 됨. 이 단순함이 GAE의 실용성의 핵심입니다(역방향으로 한 번 훑으면 계산 끝).

### 1-4. $\lambda$의 의미 — 하나의 손잡이로 Bias–Variance 조절

- $\lambda \to 0$: $\hat{A}_t = \delta_t^V$, 즉 **1-step TD**. $V$에 크게 의존 → **저분산·고편향**.
- $\lambda \to 1$: $\hat{A}_t = \sum \gamma^l \delta_{t+l}^V = \sum\gamma^l r_{t+l} - V(s_t)$, 즉 **MC**. 실제 보상 전체 사용 → **무편향·고분산**.
- 보통 $\lambda \in [0.95, 0.99]$ 사용 — MC에 가까우면서도 분산을 약간 억제하는 지점.

각 $k$-step 추정량에 부여되는 가중치는 $(1-\lambda)\lambda^{k-1}$ 입니다.
- **가까운 미래($\hat{A}^{(1)}, \hat{A}^{(2)}$)**: 큰 가중치 → 분산 낮은 신호를 많이 반영.
- **먼 미래($\hat{A}^{(\infty)}$)**: 작은 가중치 → 편향 낮은 정보를 조금씩 반영.

### 1-5. $(1-\lambda)$ 항의 역할

$(1-\lambda)$는 **정규화 상수**입니다. 가중치 합 $\sum_{k=1}^\infty (1-\lambda)\lambda^{k-1} = (1-\lambda)\cdot\frac{1}{1-\lambda} = 1$ 이 되도록 만들어, GAE가 **"가중치 합이 1인 유효한 가중평균(weighted average)"** 임을 보장합니다(강의 11쪽). 이 항이 없으면 추정치의 스케일이 왜곡됩니다.

### 1-6. 정리: GAE의 직관

GAE는 TD-error $\delta$를 시간축에 걸쳐 **"번지게(smear)"** 하는 것으로 볼 수 있습니다. 현재 가치 예측 $V$의 신뢰성과 실제 보상 $r$의 진실성 사이의 균형을, **단 하나의 하이퍼파라미터 $\lambda$** 로 조율합니다. (출처: Schulman et al., "High-Dimensional Continuous Control Using GAE", ICLR 2016)

### 1-7. ⭐ 시험 빈출 — GAE 성질 OX (강의 13, 14쪽)

| 명제 | 참/거짓 | 이유 |
|------|---------|------|
| GAE($\gamma$, 0)은 TD(0) return을 쓰는 advantage | **참** | $\lambda=0 \Rightarrow \hat{A}=\delta_t^V$ (1-step TD) |
| GAE($\gamma$, 1)은 TD(0) return을 쓰는 advantage | **거짓** | $\lambda=1$은 MC return에 해당 |
| GAE($\gamma$, .99)은 TD(0) return을 쓰는 advantage | **거짓** | $\lambda=.99$는 MC에 가까움 |
| GAE($\gamma$, 0)의 **편향**이 GAE($\gamma$, 1/.99)보다 클 가능성이 큼 | **참** | $\lambda{=}0$은 $V$에 의존 → 고편향 |
| GAE($\gamma$, 0)의 **분산**이 GAE($\gamma$, 1)보다 클 가능성이 큼 | **거짓** | $\lambda{=}0$은 저분산. 분산은 오히려 **작음** |

> 서술형 변형 대비: "$\lambda$를 0에서 1로 올리면 편향과 분산이 각각 어떻게 변하며 그 이유는?" → **편향 감소, 분산 증가**. 이유는 실제 보상 의존도가 커지고 학습된 $V$ 의존도가 줄기 때문.

### 1-8. GAE의 한계 / 확장 방향 (서술 보강용)

- **한계**: $\lambda$는 여전히 튜닝이 필요한 하이퍼파라미터이며, 가치함수 $V$의 품질에 추정 정확도가 크게 좌우됨($V$가 나쁘면 저$\lambda$ 영역에서 편향이 심해짐). $\gamma$와 $\lambda$가 함께 유효 지평(effective horizon)을 결정하므로 상호작용을 고려해야 함.
- **확장**: GAE가 만들어낸 "깨끗한 Advantage 신호"가 있어야 IS 기반 off-policy 업데이트(TRPO/PPO)가 견고해짐. 즉 GAE는 다음 주제인 TRPO/PPO의 **전제 조건**.

## 2. TRPO (Trust Region Policy Optimization)

### 2-1. 제안 배경 — 바닐라 Policy Gradient의 한계 (강의 16쪽)

목적은 $\max_\theta \eta(\pi_\theta)$ 이고, 이론적 업데이트는 $\theta_{k+1} \leftarrow \theta_k + \alpha\nabla_\theta\eta(\pi_\theta)$ 입니다. 그러나 실제 수치적 구현에는 네 가지 문제가 있습니다.

1. **1차 근사의 한계**: 추정한 경사 $\widehat{\nabla_\theta\eta}$ 는 1차 근사일 뿐 → 함수가 선형이라고 가정하며 **곡률(curvature) 정보를 못 담음**.
2. **고분산**: 상태·행동 공간이 크면 경사 추정의 분산이 큼.
3. **보폭 $\alpha$의 기준 오류**: $\alpha$는 **파라미터 공간 $\theta$** 기준이지, 우리가 정작 관심 있는 **목적함수나 정책 분포 $\pi$** 기준이 아님. 즉 같은 $\Delta\theta$라도 정책 분포는 미세하게 또는 격렬하게 변할 수 있음.
4. **보폭 딜레마**: 너무 크면 **오버슈팅 → 성능 붕괴(collapse)**, 너무 작으면 **학습이 느림**.

> 서술 포인트: 바닐라 PG의 근본 문제는 "**파라미터 공간에서의 거리 ≠ 정책 공간에서의 거리**"라는 불일치입니다. TRPO는 이 불일치를 정면으로 다룹니다.

### 2-2. Trust Region 개념 (강의 17쪽)

**신뢰 영역(trust region)** = "우리가 잘 이해하고 있다고 믿는 국소 영역". 그 영역 안에서만 최적점을 찾습니다.

$$\theta_{k+1} = \arg\max_\theta \mathcal{L}(\theta_k,\theta)\quad \text{s.t.}\quad \|\Delta\|\le\delta$$

즉 "근사가 믿을 만한 범위 안에서만 최대한 좋아지는 점으로 이동"하는 전략입니다.

### 2-3. 목적함수(Surrogate) 유도 — 어디서 IS와 KL이 나오는가

**(a) 상대 성능 항등식 → 대리 목적함수**
할인된 미래 상태 분포 $d^\pi$를 이용하면 성능 차이를 다음처럼 쓸 수 있습니다(강의 5, 22쪽).

$$J(\pi') - J(\pi) = \frac{1}{1-\gamma}\mathbb{E}_{\substack{s\sim d^{\pi'}\\a\sim\pi'}}\left[A^\pi(s,a)\right]$$

여기에 **중요도 샘플링**을 적용하면 행동 기대값을 옛 정책 $\pi$ 기준으로 바꿀 수 있습니다.

$$= \frac{1}{1-\gamma}\mathbb{E}_{\substack{s\sim d^{\pi'}\\a\sim\pi}}\left[\frac{\pi'(a|s)}{\pi(a|s)}A^\pi(s,a)\right]$$

**(b) 유일하게 남는 문제**: 상태 분포가 여전히 $s\sim d^{\pi'}$ (새 정책 기준)입니다. 새 정책으로 데이터를 모으지 않았으니 이건 곤란합니다.

**(c) 핵심 근사**: $d^{\pi'}\approx d^\pi$ 로 두고 무시합니다(강의 20쪽). 그러면 **옛 정책 $\pi$의 궤적만으로 최적화 가능한** 대리 목적함수 $\mathcal{L}_\pi(\pi')$ 가 됩니다.

$$\mathcal{L}_\pi(\pi') = \frac{1}{1-\gamma}\mathbb{E}_{\substack{s\sim d^{\pi}\\a\sim\pi}}\left[\frac{\pi'(a|s)}{\pi(a|s)}A^\pi(s,a)\right]$$

> **IS인데 왜 안 터지나(강의 22쪽 핵심)**: 일반적인 궤적 단위 IS는 매 스텝 비율의 **곱**이라 길이가 길어지면 0으로 소멸하거나 폭발합니다. 그러나 여기 가중치는 **현재 시점 한 스텝의 비율**에만 의존하고 이전 history의 곱이 없으므로 **소멸/폭발하지 않습니다.** 이게 이 대리 목적함수의 큰 장점.

### 2-4. 근사가 유효한 조건 — 상대 성능 바운드 (강의 20쪽)

$d^{\pi'}\approx d^\pi$ 근사는 두 정책이 가까울 때만 정당화됩니다. 이를 정량화한 것이 상대 성능 바운드입니다.

$$\bigl|\,J(\pi') - \bigl(J(\pi)+\mathcal{L}_\pi(\pi')\bigr)\,\bigr| \;\le\; C\sqrt{\mathbb{E}_{s\sim d^\pi}\bigl[D_{KL}(\pi'\|\pi)[s]\bigr]}$$

즉 **두 정책이 KL-발산 기준으로 가까우면, 대리 목적함수의 근사 오차가 작다**는 보장입니다. ($C$를 어떻게 정하는지는 PPO 단계에서 다룸.)

### 2-5. 단조 향상 보장(Monotonic Improvement) (강의 21, 24쪽)

위 바운드 덕분에, 대리 목적함수 $\mathcal{L}$ 를 (KL 제약 하에) 개선하면 **실제 성능 $\eta$도 절대 나빠지지 않음**이 보장됩니다.

$$\theta_{k+1} = \arg\max_\theta \mathcal{L}(\theta_k,\theta) \quad\text{s.t.}\quad \bar{D}_{KL}(\pi_\theta\|\pi_{\theta_k}) \le \delta$$
$$\Longrightarrow \quad \eta(\pi_{\theta_{k+1}}) \ge \eta(\pi_{\theta_k})$$

여기서 KL 제약은 **옛 정책이 방문한 상태들에 대한 평균 KL**입니다. "파라미터 사이의 거리가 곧 정책 사이의 거리"가 되도록 만드는 장치죠.

### 2-6. 최적화 방법 — 2차 미분과 Fisher 정보 행렬 (강의 24쪽)

제약 최적화를 실제로 풀기 위해 TRPO는:
1. KL-발산을 **2차 테일러 전개** 합니다.
2. 그 2차 항에서 **Fisher Information Matrix(FIM)** 가 등장 → 정책 공간의 **"곡률(curvature)"** 을 표현.
3. 이 FIM을 이용해 **Natural Policy Gradient(자연 경사)** 를 계산 → 신뢰 영역을 넘지 않는 선에서 개선을 최대화하는 **최적 보폭을 자동 결정**.

> 직관: 1차 경사는 "어느 방향이 오르막인가"만 알지만, FIM(2차)은 "그 방향으로 가면 정책 분포가 얼마나 빨리 변하는가"까지 알려줍니다. 그래서 정책 분포 변화량($\delta$) 기준으로 안전한 최대 보폭을 잡을 수 있습니다.

### 2-7. 핵심 혁신 (강의 24쪽)

- 바닐라 PG: **파라미터 공간**에서 보폭을 잡음.
- TRPO: **정책 공간**에서 보폭을 잡음 (KL 제약 + 2차 미분).
- 결과: 고정 학습률에서 흔히 생기는 **성능 붕괴를 방지**하고 **단조 향상을 수학적으로 보장**.

### 2-8. ⭐ PG vs TRPO 비교표 (강의 25쪽, 서술형 단골)

| 항목 | 바닐라 PG | TRPO |
|------|-----------|------|
| **업데이트 공간** | 파라미터 공간 $\theta$ — 실제 확률 변화량을 모른 채 가중치 직접 갱신 | 정책 공간 $\pi$ — 실제 확률분포 변화를 엄격히 통제 (KL 제약 + 2차 미분) |
| **보폭(step size)** | 수동·고정. 너무 크면 붕괴, 너무 작으면 느림 | 자동·적응. FIM으로 "안전한 최대 보폭"을 계산 |
| **데이터 효율** | On-policy. 한 번 경사 갱신 후 데이터 폐기 | Off-policy(IS). 신뢰 영역 내에서 데이터 다회 재사용 |
| **안정성 보장** | 없음 | 단조 향상(Monotonic Improvement) 보장 |

### 2-9. TRPO의 한계 / 확장 방향 → PPO

- **한계**: Fisher 정보 행렬을 다루는 **2차 최적화는 계산 비용이 큼**(켤레경사법 등 필요), 대규모 신경망에서 구현이 복잡하고 무거움. KL 제약을 정확히 만족시키는 절차가 번거로움.
- **확장(다음 강의로의 연결)**: **PPO**는 TRPO의 "신뢰 영역" 철학을 유지하되, 복잡한 2차 제약 대신 **목적함수의 비율 $\frac{\pi_\theta}{\pi_{old}}$ 를 클리핑(clipping)** 하는 단순한 1차 방법으로 비슷한 효과를 냅니다. "TRPO의 안정성을 더 싸고 간단하게"가 PPO의 동기입니다.

---

## 3. 두 주제를 잇는 통합 서술 (고득점 포인트)

> 강의 6~7쪽이 강조하는 핵심 연결고리입니다. 서술형에서 GAE와 TRPO를 따로 답하기보다, **하나의 이야기로 엮으면** 통합적 이해가 드러납니다.

1. **데이터 재사용의 필요** → IS로 off-policy 학습을 가능케 함.
2. **IS의 부작용** → 비율을 곱하면서 분산이 증폭됨. 완벽한 IS 보정조차 **Advantage 신호가 노이즈면 학습이 실패**.
3. **GAE의 역할** → $\lambda$로 bias–variance를 조절해 **깨끗하고 안정적인 Advantage**를 생산 → IS의 불안정성을 완화.
4. **TRPO의 역할** → IS 근사($d^{\pi'}\approx d^\pi$)가 유효하려면 정책이 가까워야 하므로, **KL 신뢰 영역**으로 보폭을 통제 → 단조 향상 보장.
5. **PPO** → GAE(신호) + TRPO 철학(안정)을 결합하되, TRPO의 2차 최적화를 클리핑으로 단순화한 실용적 종착점.

## 4. 예상 서술형 문제 & 답안 골격

**Q1. GAE가 제안된 배경을, N-step 추정량의 bias–variance 관점에서 설명하라.**
→ 핵심: 1-step TD(저분산·고편향) ↔ MC(무편향·고분산)의 양 극단. 단일 $k$ 선택의 한계. 모든 $k$를 지수가중 평균하여 양쪽 장점을 절충하려는 동기. IS와 곱해질 때 분산 증폭 문제도 함께 언급하면 가점.

**Q2. GAE 최종식 $\hat{A}_t = \sum_l (\gamma\lambda)^l\delta_{t+l}^V$ 의 유도 과정을 설명하라.**
→ ① $\hat{A}^{(k)}=\sum\gamma^l\delta_{t+l}^V$ (텔레스코핑) → ② $(1-\lambda)$ 가중 평균 전개 → ③ $\delta$별 계수 묶기 → ④ 등비급수 합 $\frac{1}{1-\lambda}$ 가 $(1-\lambda)$와 상쇄 → ⑤ $(\gamma\lambda)^l$ 가중 합으로 수렴.

**Q3. $\lambda \to 0$ 과 $\lambda \to 1$ 일 때 GAE가 각각 무엇이 되며, bias·variance는 어떻게 달라지는가?**
→ $\lambda{=}0$: TD(0), 저분산·고편향. $\lambda{=}1$: MC, 무편향·고분산. $(1-\lambda)$의 정규화 역할도 언급.

**Q4. 바닐라 PG의 보폭(step size) 문제를 설명하고, TRPO가 이를 어떻게 해결하는가?**
→ $\alpha$가 파라미터 공간 기준이라 정책 분포 변화를 통제 못 함 → 붕괴/저속 딜레마. TRPO는 KL 제약으로 정책 공간에서 보폭을 잡고, FIM(2차)으로 안전한 최대 보폭을 자동 결정 → 단조 향상 보장.

**Q5. TRPO의 대리 목적함수 $\mathcal{L}_\pi(\pi')$ 가 어떻게 유도되며, 왜 KL 제약이 필요한가?**
→ 상대 성능 항등식 → IS 적용 → $d^{\pi'}\approx d^\pi$ 근사 → 이 근사는 정책이 가까울 때만 유효(상대 성능 바운드) → KL 제약으로 근접성 강제.

**Q6. TRPO에서 IS를 쓰는데도 가중치가 폭발/소멸하지 않는 이유는?**
→ 궤적 전체 비율의 곱이 아니라 현재 시점 한 스텝의 비율에만 의존하기 때문(이전 history 곱 없음).

**Q7. TRPO의 한계와 PPO로의 발전 방향을 논하라.**
→ 2차 최적화(FIM)의 높은 계산 비용·구현 복잡성. PPO는 클리핑된 대리 목적함수로 1차 방법만으로 신뢰 영역 효과를 근사 → 더 단순·저비용.

**Q8. (통합) IS, GAE, TRPO가 어떻게 맞물려 안정적인 off-policy 정책 최적화를 이루는지 서술하라.**
→ 섹션 3의 5단계 흐름을 그대로 서술.

---

### 한 줄 요약
- **GAE** = $\lambda$ 하나로 bias–variance를 조율해 **깨끗한 Advantage(방향의 질)** 를 만든다.
- **TRPO** = KL 신뢰 영역 + 2차 미분으로 **안전한 보폭(이동의 안정)** 을 보장한다(단조 향상).
- **PPO** = 둘을 결합하되 TRPO의 무거운 2차 최적화를 클리핑으로 단순화한 실용형.
