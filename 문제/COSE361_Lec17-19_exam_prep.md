# 정답 및 해설

## Part 1 해설

**A1.**
- **파라메트릭**: 데이터 양과 무관하게 **고정된 개수의 파라미터 $\theta$** 를 가짐. 예: 선형 회귀, 로지스틱 회귀.
- **비파라메트릭**: 모델 복잡도가 데이터에 따라 증가. 예: k-NN, 결정 트리.
- **k-NN이 비파라메트릭인 이유**: 명시적 파라미터를 학습하지 않고 전체 학습 데이터를 저장(memorize)하며, 예측 시 데이터 분포에 따라 결정 경계가 암묵적으로 형성됨. 데이터가 많아질수록 "모델"(저장된 점들)의 크기가 커진다.

**A2.**
- **Train**: 모델 파라미터 학습. **Validation**: 모델 종류·하이퍼파라미터(학습률 등) 튜닝, 학습 과정이 잘 작동하는지 평가. **Test**: 최종 일반화 성능 보고(한 번만 사용).
- Test로 반복 튜닝하면 → test set 정보가 모델 선택에 **누출(leak)** 되어, test 점수가 더 이상 진짜 일반화 성능을 반영하지 못하고 낙관적으로 편향된다.
- (비유: train = 연습 문제, validation = 모의고사, test = 본 시험)

**A3.**
- (a) 검증 1 fold = **200개**, 학습 4 fold = **800개**.
- (b) **5번** 학습 (각 fold가 한 번씩 검증 역할).
- (c) **데이터가 적을 때**. 모든 데이터를 학습·검증에 활용하고, 5개 검증 점수를 평균내어 단일 분할의 운(특정 분할에 따른 변동)을 완화 → 더 강건한 성능 추정.

**A4.**
- **Underfitting**: 모델이 너무 단순 → 학습 데이터의 패턴조차 못 잡음 → **high bias**.
- **Overfitting**: 모델이 너무 복잡 → 학습 노이즈까지 암기 → 새 데이터에서 실패 → **high variance**.
- 복잡도 증가 시: **training error는 단조 감소**(점점 더 잘 맞춤), **test error는 감소하다 증가**(U자형). 처음엔 진짜 패턴을 잡아 좋아지지만, 일정 지점 이후엔 노이즈를 학습해 일반화가 나빠진다. test error 최소점이 최적 복잡도.

**A5. (유도)**
$y = f + \epsilon$ 이고 $\epsilon \perp \hat f$, $\mathbb{E}[\epsilon]=0$, $\text{Var}(\epsilon)=\sigma^2$.

$$\mathbb{E}[(y-\hat f)^2] = \mathbb{E}[(f+\epsilon-\hat f)^2] = \mathbb{E}[(f-\hat f)^2] + 2\mathbb{E}[\epsilon(f-\hat f)] + \mathbb{E}[\epsilon^2]$$

중간 항: $\epsilon$ 독립이고 $\mathbb{E}[\epsilon]=0$ → $2\,\mathbb{E}[\epsilon]\,\mathbb{E}[f-\hat f]=0$. 마지막 항: $\mathbb{E}[\epsilon^2]=\sigma^2 = \text{Noise}$. 따라서

$$= \mathbb{E}[(f-\hat f)^2] + \sigma^2$$

이제 $\pm\,\mathbb{E}[\hat f]$ 를 넣어 분해:

$$\mathbb{E}[(f-\hat f)^2] = \mathbb{E}\big[(f-\mathbb{E}[\hat f] + \mathbb{E}[\hat f]-\hat f)^2\big]$$
$$= (f-\mathbb{E}[\hat f])^2 + 2(f-\mathbb{E}[\hat f])\,\mathbb{E}[\mathbb{E}[\hat f]-\hat f] + \mathbb{E}\big[(\hat f-\mathbb{E}[\hat f])^2\big]$$

$(f-\mathbb{E}[\hat f])$ 는 상수, $\mathbb{E}[\mathbb{E}[\hat f]-\hat f]=0$ → 중간 항 0. 따라서

$$= \underbrace{(f-\mathbb{E}[\hat f])^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}[(\hat f-\mathbb{E}[\hat f])^2]}_{\text{Variance}}$$

종합하면 $\;\mathbb{E}[(y-\hat f)^2] = \text{Bias}^2 + \text{Variance} + \text{Noise}.\;\blacksquare$

**A6.**
- 단순 모델: **high bias, low variance**. 복잡 모델: **low bias, high variance**.
- **Sweet spot**: $\text{Bias}^2 + \text{Variance}(+\text{Noise})$ 인 총 오차가 최소가 되는 복잡도. 한쪽을 줄이면 다른 쪽이 커지므로 그 균형점을 찾는 것. (단 슬라이드대로 "usually" — 항상 존재하지는 않음.)

**A7.**
- **L2 (Ridge)**: $R(W)=\sum_j W_j^2$. **L1 (Lasso)**: $R(W)=\sum_j |W_j|$.
- **L1 희소성**: L1의 제약 영역은 **마름모(다이아몬드)**. 데이터 손실의 등고선이 이 영역과 처음 닿는 점이 대개 **꼭짓점(축 위)** 이 되어 일부 가중치가 정확히 0이 된다. 반면 L2의 원형 제약은 축 위에서 닿을 확률이 낮아 0이 잘 만들어지지 않는다.

**A8.**
- $\lambda \to 0$: 정규화 없는(unregularized) 문제와 동일 → 가중치가 자유롭게 커질 수 있음 → 과대적합 위험.
- $\lambda$ 매우 큼: 가중치가 강하게 작은 값(0 쪽)으로 눌림 → 모델 단순화 → 과소적합 위험.

**A9.**
- (a) **과대적합(overfitting)**: train은 잘 맞지만 일반화 실패 → **high variance**.
- (b) (택 3) 정규화(L1/L2) 추가, 모델 복잡도 감소(더 단순한 모델), 학습 데이터 추가, (트리면) 가지치기, 교차검증으로 하이퍼파라미터 조정, early stopping/dropout 등.

**A10.**
- 학습 데이터를 완벽히 암기(룩업 테이블)해도 **본 적 없는 입력**에 대해선 아무 정보도 주지 못한다. 일반화 = 같은 분포에서 뽑힌 **새 데이터**에 대한 성능. 룩업 테이블은 새 입력에 일반화하지 못하므로 좋은 모델이 아니다.

## Part 2 해설

**A11.**
- **분류**: 쿼리의 $k$개 최근접 이웃의 **다수결(mode)** 로 레이블 결정. **회귀**: $k$개 이웃의 **타겟 평균**.
- "학습 단계 없음": 모델을 따로 만들지 않고 데이터를 **저장만** 함(lazy learning). 모든 계산은 예측 시점에 일어난다.

**A12.**
거리 계산 ($q=(2,3)$):

| 점 | 계산 | 거리 |
|----|------|------|
| C=(3,4) | $\sqrt{1+1}$ | $\sqrt2 \approx 1.414$ |
| B=(2,1) | $\sqrt{0+4}$ | $2.0$ |
| E=(0,3) | $\sqrt{4+0}$ | $2.0$ |
| A=(1,1) | $\sqrt{1+4}$ | $\sqrt5 \approx 2.236$ |
| D=(5,5) | $\sqrt{9+4}$ | $\sqrt{13} \approx 3.606$ |

- (a) $k=1$: 가장 가까운 C → **Blue**
- (b) $k=3$: {C, B, E} = {Blue, Red, Red} → 다수결 **Red**
- (c) $k=1$은 가장 가까운 한 점에만 의존 → **low bias, high variance**(노이즈에 민감, 경계가 복잡). $k$를 키우면 더 많은 이웃을 투표에 반영 → 더 부드럽고 안정적(**high bias, low variance**). 여기선 $k$를 늘리자 인접한 Red 점들의 영향으로 예측이 뒤집혔다.

**A13.** 차이 벡터 $= (-2, 3, -3)$.
- (a) L2 $= \sqrt{(-2)^2+3^2+(-3)^2} = \sqrt{4+9+9} = \sqrt{22} \approx 4.69$
- (b) L1 $= |{-2}|+|3|+|{-3}| = 2+3+3 = 8$
- (c) Minkowski $p{=}3 = (2^3+3^3+3^3)^{1/3} = (8+27+27)^{1/3} = 62^{1/3} \approx 3.96$

**A14.**
- **작은 $k$ ($k=1$)**: low bias, high variance. 노이즈/이상치에 매우 민감. 결정 경계가 복잡·들쭉날쭉.
- **큰 $k$**: high bias, low variance. 결정 경계 부드러움. 노이즈에 둔감하나 경계 근처 점을 오분류할 수 있음.
- **홀수 $k$**: 이진 분류에서 동표(tie)를 피하기 위해.

**A15.**
- **문제**: 거리 계산에서 범위가 훨씬 큰 **소득이 거리를 지배** → 나이 특성이 사실상 무시됨.
- **표준화(Z-score)**: $\dfrac{X-\mu}{\sigma}$, 평균 0·표준편차 1. 고정 범위 없음, 이상치에 덜 민감.
- **정규화(Min-Max)**: $\dfrac{X-\min(X)}{\max(X)-\min(X)}$, [0,1] 범위. 범위 고정, 이상치에 민감.

**A16.**
- 차원이 커지면 특성 공간이 광대해지고 유한한 데이터는 상대적으로 **희소**해진다. 그 결과 점들이 서로 멀어지고 **"거의 등거리"** 가 되어 "가장 가까운 이웃"이라는 개념이 의미를 잃는다 → k-NN 성능 저하.

**A17.**
- **Root node**: 전체 데이터, 시작점. **Decision node**: 특성 테스트(예: Outlook=Sunny?). **Branch**: 테스트 결과(Yes/No 등). **Leaf node**: 최종 예측(클래스 또는 연속값).
- **Greedy**: 각 단계에서 당장 가장 좋아 보이는 분할 선택. **Recursive**: 같은 분할 과정을 각 자식 노드에 반복 적용.
- **전역 최적 미보장**: 매 단계 국소적으로만 최선을 택하므로, 지금은 덜 좋아 보여도 나중에 더 좋은 트리로 이어질 분할을 놓칠 수 있다.

**A20.**
- **엔트로피**: 정보이론 기반, 노드의 불확실성. 범위 $0$(순수) ~ $\log_2 C$(최대 불순도).
- **Gini**: 무작위로 레이블링한 샘플을 오분류할 확률. 범위 $0$(순수) ~ $1-\tfrac{1}{C}$(최대 불순도).
- **순수 노드**: 둘 다 **0**. 둘 다 클래스가 고르게 섞일수록 값이 커진다. 실무에선 비슷한 분할 결과를 주지만 Gini가 로그 계산이 없어 더 빠르다.

**A21.**
- **과대적합**: 트리가 깊어지며 노이즈·특정 패턴까지 학습 → train 완벽, test 나쁨(암기).
- **Pre-pruning(조기 종료)**: 자라는 도중 멈춤 — 최대 깊이, 최소 분할 샘플 수, 최소 leaf 샘플 수, 최소 불순도 감소 등. 빠르지만 너무 일찍 멈출 위험.
- **Post-pruning**: 완전히 키운 뒤 설명력 낮은 가지를 잘라냄. 더 효과적이나 계산 비용이 크다.
- **$\alpha$**: 복잡도(leaf 수 등)에 대한 페널티 강도. $\alpha$가 클수록 더 단순한 트리를 선호.

**A22.**
- **장점**: 이해/해석 쉬움(시각화 가능), 수치·범주형 모두 처리, 비파라메트릭(분포 가정 없음), 암묵적 특성 선택(중요 특성이 위쪽에).
- **단점**: 가지치기 없으면 과대적합, 불안정, 클래스 불균형 시 편향, 결정 경계가 축에 평행, greedy라 전역 최적 미보장.
- **Axis-parallel 한계**: 각 분할이 한 특성 기준 임계값이라 경계가 축에 평행한 **계단 모양** → 대각선 경계를 잘 못 표현(많은 분할 필요).
- **Unstable**: 데이터의 작은 변화가 루트 분할을 바꾸면 전체 트리 구조가 크게 달라질 수 있다.

## Part 3 해설

**A23.**
- **지도**: $(x,y)$ 레이블 쌍으로 입력→출력 매핑 학습. 회귀/분류. 예: 선형·로지스틱 회귀, 결정 트리.
- **비지도**: 레이블 없는 데이터($x$만)에서 내재 구조 발견. 예: 군집화, 차원 축소, 이상 탐지.

**A24.**
- 단계: ① $k$ 지정 → ② 무작위로 $k$개 점을 초기 중심으로 → ③ 각 점을 가장 가까운 중심에 배정(유클리드) → ④ 각 중심을 배정된 점들의 평균으로 갱신 → ⑤ 중심이 거의 안 움직이거나 최대 반복까지 ③④ 반복.
- 목적 함수: $\;WCSS = \sum_{i=1}^{k}\sum_{x\in C_i}\|x-\mu_i\|^2\;$ (inertia).
- **지역 최적만 보장**: 각 반복은 WCSS를 감소(또는 유지)시키지만, 초기화에 따라 서로 다른 지역 최소에 수렴할 수 있어 전역 최소를 보장하지 못한다.

**A25. (k-Means 계산)**
- (a) 배정 ($c_1=(1,1),\ c_2=(5,4)$):
  - $P_1$: $d(c_1)=0$ vs $d(c_2)=5$ → **군집1**
  - $P_2$: $d(c_1)=1$ vs $d(c_2)=\sqrt{18}\approx4.24$ → **군집1**
  - $P_3$: $d(c_1)=\sqrt{13}\approx3.61$ vs $d(c_2)=\sqrt2\approx1.41$ → **군집2**
  - $P_4$: $d(c_1)=5$ vs $d(c_2)=0$ → **군집2**
  - 군집1 $=\{P_1,P_2\}$, 군집2 $=\{P_3,P_4\}$
- (b) 갱신: $c_1=\big(\tfrac{1+2}{2},\tfrac{1+1}{2}\big)=(1.5,\,1)$, $\;c_2=\big(\tfrac{4+5}{2},\tfrac{3+4}{2}\big)=(4.5,\,3.5)$
- (c) WCSS:
  - 군집1: $(0.25+0)+(0.25+0)=0.5$
  - 군집2: $(0.25+0.25)+(0.25+0.25)=1.0$
  - **WCSS $= 1.5$**

**A26.**
- 초기 중심 위치에 따라 최종 군집(지역 최적)이 달라진다. 나쁜 초기화는 나쁜 군집/느린 수렴을 부른다.
- **k-Means++**: 첫 중심을 무작위로 뽑고, 이후 중심은 이미 뽑힌 중심들로부터 **멀리 떨어진 점일수록 높은 확률**로 뽑아 중심들을 통계적으로 멀리 분산 → 더 좋은 초기화.
- **확률을 쓰는 이유**: 무조건 가장 먼 점을 뽑으면 **이상치(outlier)** 를 중심으로 택할 위험이 크다. $D(x)^2$에 비례하는 확률로 뽑으면 멀리 있는 점을 선호하되 이상치를 항상 고르지는 않게 된다.

**A27.**
- **엘보우**: $k$를 1~10 등으로 바꿔가며 WCSS 계산 → $k$ vs WCSS 그래프에서 감소율이 급격히 꺾이는 **"팔꿈치"** 지점을 선택 (즉 $\tfrac{WCSS(k-1)-WCSS(k)}{WCSS(k-1)} < \text{threshold}$).
- WCSS는 $k$가 커지면 단조 감소($k=n$이면 0)하지만, 너무 크면 **과분할**되어 무의미. 엘보우 이후엔 추가 군집이 주는 WCSS 감소 이득이 미미하므로 그 지점에서 멈춘다.

**A28.**
- **한계**: $k$를 미리 지정해야 함, 초기화·이상치에 민감, **구형(spherical) 블롭** 가정 → 비볼록 형태나 밀도가 다른 군집에서 실패, 고차원에서 거리 신뢰도 하락.
- **실패 분포**: 동심원, 초승달(반달) 모양, 밀도 차가 큰 군집 등.
- **밀도 기반 필요 이유**: 중심(centroid)이 아니라 **밀도**로 군집을 정의하면 임의 형태 군집과 노이즈를 다룰 수 있다 → DBSCAN.

**A29.**
- **$\varepsilon$**: 한 점의 이웃을 정의하는 반경. **MinPts**: core가 되기 위해 $\varepsilon$ 내에 필요한 최소 점 수(자신 포함).
- **Core**: $\varepsilon$ 내 이웃 ≥ MinPts. **Border**: 자신은 core가 아니지만 어떤 core의 $\varepsilon$ 내에 있음. **Noise**: core도 border도 아님.
- 분류:
  - $P$ (이웃 5 ≥ 4) → **Core**
  - $Q$ (이웃 2 < 4 이지만 core $P$의 $\varepsilon$ 내) → **Border**
  - $R$ (이웃 1, 어떤 core의 $\varepsilon$ 밖) → **Noise**

**A30.**
- **DBSCAN 장점**: $k$ 지정 불필요 / 임의 형태 군집 발견 / 이상치(노이즈) 자연 식별.
- **단점**: $\varepsilon$·MinPts 선택에 민감 / 밀도 차가 큰 군집에서 곤란(하나의 $\varepsilon$로 모두 못 맞춤). (+고차원에서 거리 신뢰도 하락)
- **$k$ 불필요 이유**: 군집을 **밀도(연결된 dense 영역)** 로 정의하므로 군집 수가 데이터로부터 자동 결정된다.

**A31.**
- **과대적합 위험↑**: 특성/샘플 비가 커져 모델이 학습 데이터 특유의 노이즈를 학습.
- **계산 복잡도↑**: 많은 알고리즘이 차원이 늘수록 시간/메모리 측면에서 성능이 나빠짐.
- **데이터 요구량↑**: 같은 데이터 밀도를 유지하려면 차원이 늘수록 필요한 데이터가 **지수적**으로 증가.

**A32.**
- **특성 선택**: 원래 특성의 **부분집합**을 고름. 새 특성 안 만듦, 해석 가능성 유지.
- **특성 추출**: 원래 특성들의 (선형) 결합으로 더 적은 **새 특성** 생성. 새 특성은 직접 해석이 어려울 수 있음(예: PCA).
- **Filter**: 중복/무의미 특성 제거(예: 높은 상관 특성). **Wrapper**: 성능 좋은 특성 부분집합을 탐색. **Embedded**: 학습 과정에서 선택(예: L1 정규화, 트리 특성 중요도).

**A33.**
- **목표**: 데이터의 분산을 최대로 담는 새 **직교 축(주성분)** 을 찾아 차원 축소.
- **직관**: 데이터에 선을 적합하면 그게 PC1(분산 최대). PC2는 PC1에 직교하며 다음으로 분산이 큼. 상위 $k$개를 택해 차원 축소.
- **주성분 = 공분산 행렬의 고유벡터(eigenvector)**. **고유값(eigenvalue)** = 그 주성분 방향으로 투영했을 때의 **분산**(=그 방향이 담는 분산의 크기/중요도).

**A34. (PCA 계산)**
- (a) $\det(C-\lambda I)=0$: $(4-\lambda)^2-4=0 \Rightarrow \lambda^2-8\lambda+12=0 \Rightarrow \lambda=\tfrac{8\pm\sqrt{16}}{2}$ → $\boldsymbol{\lambda_1=6,\ \lambda_2=2}$.
- (b) $\lambda_1=6$: $(C-6I)v=0 \Rightarrow -2v_1+2v_2=0 \Rightarrow v_1=v_2$ → $\mathbf{v_1=\tfrac{1}{\sqrt2}(1,1)}$.
  $\lambda_2=2$: $(C-2I)v=0 \Rightarrow v_1+v_2=0 \Rightarrow v_1=-v_2$ → $\mathbf{v_2=\tfrac{1}{\sqrt2}(1,-1)}$.
- (c) **PC1** $=$ 더 큰 고유값 $\lambda_1=6$ 에 대응하는 $\tfrac{1}{\sqrt2}(1,1)$. 설명 분산 비율 $= \dfrac{\lambda_1}{\lambda_1+\lambda_2} = \dfrac{6}{8} = \mathbf{0.75\ (75\%)}$.

**A35.**
- 단계: ① 데이터 **표준화**(평균0·분산1) → ② 공분산 행렬 $C$ 계산 → ③ $C$의 고유벡터·고유값 계산 → ④ 고유값 내림차순으로 고유벡터 정렬 → ⑤ 상위 $k$개 고유벡터 선택 → ⑥ 표준화 데이터를 선택된 주성분에 투영 ($X_{new}=X_{scaled}\,W$).
- **표준화 이유**: PCA는 스케일에 민감하다. 스케일이 큰 특성이 분산을 지배해 그 방향이 주성분이 되어버리므로, 모든 특성을 동일 스케일로 맞춰야 공정하다.
- **$k$ 선택**: ① **누적 설명 분산 비율** 85–95%가 되는 $k$. ② **Scree plot**(고유값 vs 성분 인덱스)에서 "팔꿈치". (③ 시각화 목적이면 $k=2$ 또는 $3$.)

### 시험 직전 핵심 체크리스트
- **계산이 바로 나와야 하는 것**: Gini / Entropy 공식, MSE Reduction, WCSS, 공분산행렬 고유값·설명분산비율, 거리(L1/L2/Lp)
- **자주 헷갈리는 포인트**: bias↔variance ↔ under/overfitting ↔ 모델 복잡도의 대응 관계, L1(희소·다이아몬드) vs L2(원), Standardization vs Min-Max, Core/Border/Noise 판정 기준
- **"왜?"를 묻는 단골**: k-Means가 지역 최적인 이유, k-Means++가 확률을 쓰는 이유, DBSCAN이 $k$ 불필요한 이유, PCA 전 표준화하는 이유, 결정 트리가 unstable·axis-parallel인 이유
